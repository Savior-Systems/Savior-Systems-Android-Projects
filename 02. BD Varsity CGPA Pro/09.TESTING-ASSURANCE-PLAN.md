## 1. CGPA Calculation Accuracy Tests
`
These are the highest-priority tests. A CGPA calculator that produces wrong results is worse than no calculator at all.
`
### Test Case CA-01: Single Semester, Single Course
*   **Input**: 1 semester, 1 course: "CSE101", 3.0 credits, grade A+ (4.00, DU scale)
*   **Expected**: Semester GPA = 4.00, Cumulative CGPA = 4.00, Total Credits = 3.0
*   **Calculation**: (4.00 × 3.0) / 3.0 = 4.00 ✓
`
### Test Case CA-02: Single Semester, Multiple Courses (Mixed Grades)
*   **Input**: 1 semester, DU scale:
    | Course | Credits | Grade | GP |
    |:-------|:--------|:------|:---|
    | CSE101 | 3.0 | A+ | 4.00 |
    | MAT102 | 3.0 | B+ | 3.25 |
    | PHY103 | 3.0 | A | 3.75 |
    | ENG104 | 2.0 | B | 3.00 |
*   **Expected**: GPA = (12.0 + 9.75 + 11.25 + 6.0) / (3+3+3+2) = 39.0 / 11.0 = **3.55**
`
### Test Case CA-03: Multi-Semester Cumulative CGPA
*   **Input**: 2 semesters, DU scale:
    *   Semester 1: 4 courses, 12 credits, weighted points = 39.0 (GPA 3.25)
    *   Semester 2: 3 courses, 9 credits, weighted points = 33.75 (GPA 3.75)
*   **Expected**: CGPA = (39.0 + 33.75) / (12 + 9) = 72.75 / 21 = **3.46**
`
### Test Case CA-04: Perfect Score (All A+)
*   **Input**: 3 semesters, 5 courses each, all A+ (4.00), 3.0 credits each
*   **Expected**: CGPA = **4.00** exactly (no floating-point drift)
`
### Test Case CA-05: All Failing Grades
*   **Input**: 1 semester, 4 courses, all F (0.00), varying credits
*   **Expected**: CGPA = **0.00**
`
### Test Case CA-06: University-Specific Grading Accuracy
*   **Input**: Same courses but different universities:
    | University | Grade B+ Means | GP |
    |:-----------|:---------------|:---|
    | DU | 65-69% | 3.25 |
    | NSU | 87-89% | 3.30 |
    | AIUB | 80-84% | 3.50 |
*   **Expected**: Selecting different university changes the available grades and GP values in the dropdown. CGPA recalculates with the new scale.
`
### Test Case CA-07: Varying Credit Hours
*   **Input**: 1 semester, DU scale:
    | Course | Credits | Grade | GP |
    |:-------|:--------|:------|:---|
    | Lab | 1.5 | A+ | 4.00 |
    | Theory | 3.0 | B | 3.00 |
    | Project | 4.5 | A- | 3.50 |
*   **Expected**: GPA = (6.0 + 9.0 + 15.75) / (1.5 + 3.0 + 4.5) = 30.75 / 9.0 = **3.42**
`
### Test Case CA-08: Decimal Precision
*   **Input**: Courses that produce a CGPA like 3.456789...
*   **Expected**: Display rounds to 2 decimal places: **3.46** (standard rounding)
`

## 2. UI/UX Functional Tests
`
### Test Case UI-01: Orientation Change Preserves Data
*   **Action**: Enter 2 semesters with courses, rotate device
*   **Expected**: All entered data (course names, credits, grades) persisted. CGPA unchanged.
*   **Note**: App is locked to portrait in v1 — verify `screenOrientation` is set.
`
### Test Case UI-02: Empty State Rendering
*   **Action**: Launch calculator with no semesters
*   **Expected**: Empty state message displayed: "Tap + to add your first semester and start calculating your CGPA"
`
### Test Case UI-03: Semester Expand/Collapse
*   **Action**: Tap semester header
*   **Expected**: Course list collapses/expands with smooth animation (300ms)
`
### Test Case UI-04: Course Deletion
*   **Action**: Tap delete icon on a course → confirm dialog → delete
*   **Expected**: Course card animates out. CGPA recalculates. If last course in semester, empty state shows.
`
### Test Case UI-05: Semester Deletion
*   **Action**: Long-press semester → Delete → confirm
*   **Expected**: Semester and all courses removed. CGPA recalculates.
`
### Test Case UI-06: Dark Mode Visual Verification
*   **Action**: Toggle dark mode in Settings
*   **Expected**: All screens render correctly. No hardcoded black/white text. Primary green adjusts to `0xFF58D68D`.
`
### Test Case UI-07: Bangla Language Switch
*   **Action**: Change language to Bangla in Settings
*   **Expected**: All UI text switches to Bangla. University names show Bangla variants. Grade labels remain in English letters (A+, B, etc.).
`
### Test Case UI-08: Grade Dropdown Completeness
*   **Action**: Select DU, add course, open grade dropdown
*   **Expected**: Dropdown shows all 10 grades: A+ through F. Each shows letter + GP (e.g., "A+ (4.00)").
`
### Test Case UI-09: University Search
*   **Action**: Open university selector → type "BRAC"
*   **Expected**: Filter shows only "BRAC University". Typing "বুয়েট" shows BUET (Bangla name search).
`
### Test Case UI-10: Share Result
*   **Action**: Calculate CGPA → tap Share
*   **Expected**: System share sheet opens with text: "My CGPA: 3.67 (DU Scale) — 6 Semesters, 120.0 Credits — BD Varsity CGPA Pro"
`

## 3. Edge Case Tests
`
### Test Case EC-01: Zero Credit Hours
*   **Input**: User tries to enter 0 or negative credit hours
*   **Expected**: Inline validation error: "Credit hours must be between 0.5 and 6.0". Course excluded from calculation.
`
### Test Case EC-02: No Grade Selected
*   **Input**: Course with credit hours but no grade chosen
*   **Expected**: Course excluded from calculation. Subtle indicator: "Select a grade". CGPA calculated from other courses only.
`
### Test Case EC-03: Maximum Courses Stress Test
*   **Input**: 8 semesters × 10 courses each = 80 courses
*   **Expected**: App remains responsive. Scroll performance ≥ 60 FPS. CGPA calculation completes < 100ms.
`
### Test Case EC-04: Very Long Course Names
*   **Input**: Course name with 100+ characters
*   **Expected**: Text truncates with ellipsis in the card. Full name visible in edit mode.
`
### Test Case EC-05: University Switch Mid-Calculation
*   **Input**: Enter courses with DU grades → change university to NSU
*   **Expected**: Warning dialog appears. If confirmed, all entered grades are cleared. New dropdown shows NSU's grading scale.
`
### Test Case EC-06: Custom University Creation
*   **Input**: Create custom university with 8 grades, non-standard GPs (e.g., A = 5.00)
*   **Expected**: Custom university appears in selector. Calculator uses custom scale. Saved in Room DB.
`
### Test Case EC-07: History Persistence After App Kill
*   **Input**: Save a result → force-kill app → reopen → check history
*   **Expected**: Saved result appears in History screen with correct timestamp and CGPA.
`
### Test Case EC-08: Rapid Input Changes
*   **Input**: Rapidly change grades back and forth (10+ changes in 5 seconds)
*   **Expected**: No ANR. No crash. CGPA display settles to correct final value.
`

## 4. Ad Integration Tests
`
### Test Case AD-01: Banner Ad Loading
*   **Action**: Open calculator screen
*   **Expected**: Banner ad loads at bottom. Content scrollable above the banner. No overlap with FAB.
`
### Test Case AD-02: Interstitial Cooldown
*   **Action**: Save result → wait 10 seconds → save again
*   **Expected**: Interstitial shown on first save (if loaded). Second save within 180s → no interstitial, just snackbar.
`
### Test Case AD-03: Ad Failure Graceful Handling
*   **Action**: Toggle airplane mode → save result
*   **Expected**: Ad fails to load silently. Snackbar "Saved!" appears without delay. No crash.
`
### Test Case AD-04: Test Ad Unit IDs in Debug
*   **Action**: Run debug build → check logcat
*   **Expected**: Logcat shows test ad unit IDs loading. No production IDs present.
