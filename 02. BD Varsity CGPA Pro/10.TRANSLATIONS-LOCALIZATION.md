# 10. Translations & Localization
```
This document defines the complete localization strategy for **BD Varsity CGPA Pro**, supporting English (default) and Bangla (বাংলা) as the secondary language.
```
---
```
## 1. Language Support
```
| Language | Code | Status | Coverage |
|:---------|:-----|:-------|:---------|
| English | `en` | Default | 100% |
| Bangla | `bn` | Secondary | 100% |
```
### Runtime Language Switching
*   Language is selectable in Settings → Language toggle
*   Selection persisted in DataStore (`language` = "en" or "bn")
*   App recreates activities on language change using `AppCompatDelegate` or `Compose` locale override
*   University names have dedicated `nameBn` fields in the database
```

```
## 5. Number Formatting
```
| Setting | English | Bangla |
|:--------|:--------|:-------|
| CGPA Display | 3.67 | 3.67 (Latin digits) |
| Credit Hours | 3.0 | 3.0 (Latin digits) |
| Grade Point | 4.00 | 4.00 (Latin digits) |
| Semester Count | 6 | ৬ (Bangla digit in labels only) |
| Date/Time | Standard locale formatting | Standard locale formatting |
```
**Decision**: CGPA, credit hours, and grade points always display in **Latin (Western) digits** regardless of language setting, because students universally reference academic numbers in Latin digits. Bangla digits (০১২৩৪৫৬৭৮৯) are used only in general UI labels where natural (e.g., "সেমিস্টার ৬").
```

```
## 6. RTL Considerations
```
Bangla is a **left-to-right (LTR)** script. No RTL layout adjustments are required.
*   `android:supportsRtl="true"` is still set in the manifest for future Arabic localization.
*   Use `start`/`end` instead of `left`/`right` for all layout properties.
