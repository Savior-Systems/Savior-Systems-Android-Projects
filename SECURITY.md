# Security Policy — Savior Systems Android Portfolio

Savior Systems is committed to providing secure, private, and offline-first mobile utilities. This document outlines our policy for reporting and addressing security vulnerabilities.

---

## 1. Supported Versions

We actively maintain and provide security patches for the following versions:

| Version | Supported |
| :--- | :--- |
| **v1.x.x** |  Active Support |
| **< v1.0.0** |  Unsupported / Development Only |

---

## 2. Vulnerability Disclosure & Reporting Process

If you discover a security vulnerability (e.g. data leak, ad SDK hijack, storage exploit) in any of our 30 applications, please **do not open a public Github Issue**. Instead, report it privately through the following steps:

1.  **Email Us**: Send a detailed report to `security@saviorsystems.com`.
2.  **Required Information**:
    *   Name and package of the affected application (e.g. `FocusPulse Timer`, `com.saviorsystems.focuspulse`).
    *   Description of the vulnerability and its potential impact.
    *   Detailed step-by-step instructions to reproduce the vulnerability (proof of concept code or screenshots are highly appreciated).
3.  **Triage Timeline**: We aim to acknowledge receipt of the report within **48 hours** and provide a resolution or patch timeline within **7 business days**.
