# 02. UI/UX Design System

## 1. Design Philosophy
The app should feel like a premium piece of hardware. A "Digital Studio" aesthetic. It must clearly communicate when it is recording vs when it is paused.

## 2. Color Palette Strategy
- **Primary Color:** Crimson Red (`#E63946`) - Used exclusively for the "Record" state and button.
- **Background (Dark Mode Preferred):** Charcoal Black (`#121212`) with subtle dark gray cards (`#1E1E1E`).
- **Accent:** Teal (`#2A9D8F`) - Used for the "AI Summarize" button and playback progress.
- **Text:** Ghost White (`#F8F9FA`).

## 3. Typography
- **Timers/Counters:** `Roboto Mono` (Monospaced font is crucial so the timer digits don't jump around).
- **Headings & Body:** `Inter`.

## 4. Component Guidelines (For Developers)
- **The Record Button:** A massive floating action button (FAB) in the center bottom. When idle, it is a Red Circle. When recording, it morphs into a Red Square (Stop).
- **Waveform Visualizer:** For MVP, if real-time FFT (Fast Fourier Transform) waveforms are too complex, use a simulated pulsing animation tied to `MediaRecorder.maxAmplitude` to show active microphone input.
- **List Items:** The dashboard list should show the Title, Date, Duration (e.g., 14:32), and an "AI Sparkle" icon if a summary exists.
