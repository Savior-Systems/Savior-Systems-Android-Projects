# 12. Logging & Analytics

## 1. Critical Events to Track (Firebase)
1. `recording_started`
2. `recording_saved` (Parameter: `duration_seconds`)
3. `ai_summary_requested`
4. `ai_summary_success` (Parameter: `duration_seconds`)
5. `ai_summary_failed` (Parameter: `error_code`)

## 2. Crashlytics for MediaRecorder
- Wrap `MediaRecorder.prepare()` and `MediaRecorder.start()` in `try-catch` blocks.
- If another app is using the microphone, Android throws an `IllegalStateException`. Log this non-fatal exception to Crashlytics so you know if users are having hardware conflicts.
