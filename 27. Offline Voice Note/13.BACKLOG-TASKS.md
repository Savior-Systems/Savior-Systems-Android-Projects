# 13. Step-by-Step Implementation Guide (Rookie Roadmap)

**Dear Rookie Developer,**
`MediaRecorder` and `ForegroundService` are two of the most complex APIs in Android because of lifecycle crashes. Follow this strictly.

### Phase 1: Foundation (Day 1)
1. Setup Project, Hilt, Room, and Navigation.
2. Add `RECORD_AUDIO` and `FOREGROUND_SERVICE_MICROPHONE` permissions to Manifest.
3. Create the `VoiceNoteEntity` and Room DAO.

### Phase 2: The Foreground Service (Day 2-3) [CRITICAL]
1. Create `VoiceRecorderService`.
2. Inside `onStartCommand`, build an Android Notification. Call `startForeground(1, notification)`.
3. Create a helper class `AudioRecorderImpl` inside the service.
4. **The MediaRecorder Flow:**
   - `recorder = MediaRecorder()`
   - `recorder.setAudioSource(MediaRecorder.AudioSource.MIC)`
   - `recorder.setOutputFormat(MediaRecorder.OutputFormat.MPEG_4)`
   - `recorder.setAudioEncoder(MediaRecorder.AudioEncoder.AAC)`
   - `recorder.setOutputFile(absoluteFilePath)`
   - `recorder.prepare()`
   - `recorder.start()`
5. Test that you can record while the screen is locked.

### Phase 3: The UI (Day 4)
1. Build `DashboardScreen` showing the list of recordings from Room.
2. Build the giant Record FAB. When tapped, send an `Intent` to start the Service.
3. Build the `PlaybackScreen` using Android's `MediaPlayer` to play the saved `.m4a` file.

### Phase 4: Vertex AI SDK (Day 5)
1. Add Firebase to the project. Add `firebase-vertexai` dependency.
2. Implement the `GeminiSummarizer` class.
3. Hook the Rewarded Ad to the "Generate Summary" button. If the user finishes the ad, trigger the Gemini SDK passing the audio file.
4. Save the returned text to Room. Update UI.
