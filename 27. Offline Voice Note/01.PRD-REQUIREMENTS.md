# 01. Product Requirements Document (PRD)

## 1. Project Context & Vision
**App Name:** Offline Voice Note Recorder  
**Category:** Productivity  
**Core Vision:** An extremely reliable, privacy-focused offline audio recorder. Users can record unlimited audio memos, lectures, and dictations fully offline. The standout feature is a "Level 3 AI Integration" using Vertex AI (Gemini 1.5 Flash) that allows users to optionally generate smart summaries and transcriptions of their recordings.

## 2. Target Audience
- Students recording long university lectures.
- Journalists conducting interviews.
- Professionals dictating meeting notes.
- Privacy-conscious users who want their audio to remain strictly on their device until they explicitly ask for an AI summary.

## 3. Scope of MVP (Minimum Viable Product)
### In-Scope Features:
1. **Core Audio Engine:** Reliable recording using Android's `MediaRecorder` API (encoded as compressed `.m4a` files).
2. **Foreground Service:** Recording must persist even if the app is swiped away, using a persistent notification.
3. **Playback Engine:** Native audio playback with seek bar.
4. **Vertex AI Smart Summary:** A button to pass the local `.m4a` file directly to the Vertex AI Firebase SDK. The model will return a text transcription and a bulleted summary.
5. **Offline Library:** Room Database storing metadata (Title, Duration, Date, FilePath) and the generated AI Summaries.

### Out-of-Scope (Future Updates):
- Cloud syncing of actual audio files to Google Drive.
- Multi-track audio editing (cutting/trimming).
- Real-time live transcription (We use post-processing transcription instead to save battery).

## 4. Rookie Developer Guidelines
- **Microphone Permissions:** You must handle `RECORD_AUDIO` permissions gracefully. If denied, show a friendly rationale.
- **State Machines:** `MediaRecorder` is a strict state machine. Calling `stop()` before `start()` will crash the app. Strictly follow the step-by-step guide.
