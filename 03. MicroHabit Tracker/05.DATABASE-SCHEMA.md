# 05. Database Schema & Storage — MicroHabit Tracker
`
This document details the SQLite schemas used in the Room Database, including data indexes and preferences mappings.
`
---
`
## 1. Room Database Tables
`
### 1.1 Habit Table (`habits_table`)
Holds metadata and active streak counts for each micro-habit.
`
| Field Name | Data Type | Key Type | Notes |
| :--- | :--- | :--- | :--- |
| `id` | INTEGER | Primary Key (Auto) | Unique identity. |
| `name` | TEXT | Non-Null | Habit title (e.g. "Read 1 Page"). |
| `streak_count` | INTEGER | Non-Null | Active streak tally. |
| `max_streak` | INTEGER | Non-Null | Highest streak milestone. |
| `last_completed` | INTEGER | Nullable | Epoch millisecond timestamp of last complete. |
| `is_frozen` | INTEGER | Non-Null | Boolean flag (0 = active, 1 = frozen). |
`
### 1.2 Completion Log Table (`completion_logs`)
Holds daily logs of when each habit was completed. Essential for calendar rendering.
`
| Field Name | Data Type | Key Type | Notes |
| :--- | :--- | :--- | :--- |
| `log_id` | INTEGER | Primary Key (Auto) | Unique log identity. |
| `habit_id` | INTEGER | Foreign Key | Reference to `habits_table(id)`. |
| `completed_date` | TEXT | Non-Null | ISO date format: `YYYY-MM-DD`. |
| `timestamp` | INTEGER | Non-Null | Exact epoch millisecond timestamp. |
`
---
`
## 2. Indices & Cascading Deletes
*   **Foreign Key Constraint:** `FOREIGN KEY(habit_id) REFERENCES habits_table(id) ON DELETE CASCADE`.
*   **Index:** An index is created on `completion_logs(habit_id)` to optimize search queries during calendar rendering.
`
---
`
## 3. DataStore Preferences
Preferences are managed in local key-value DataStore sheets.
`
*   `theme_mode`: INTEGER (0 = System, 1 = Light, 2 = Dark).
*   `has_completed_onboarding`: BOOLEAN.
*   `available_freezes`: INTEGER (Count of available freeze tokens).
*   `last_sync_timestamp`: INTEGER (Epoch timestamp of last successful Firestore sync).
`
