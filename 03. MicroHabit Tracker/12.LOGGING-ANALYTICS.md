# 12. Logging & Analytics — MicroHabit Tracker

This document defines the Firebase Analytics telemetry schema and custom event parameters logged for app optimization.

---

## 1. Custom Firebase Events

To evaluate feature engagement and user retention without collecting personally identifiable information (PII), we log the following custom events:

### 1.1 `habit_created`
Triggered whenever a user creates a new micro-habit.
*   **Parameters:**
    *   `habit_category`: String (e.g. "productivity", "health")
    *   `reminder_set`: Integer (0 = no reminder, 1 = reminder set)

### 1.2 `habit_completed`
Triggered when a habit checkoff swipe occurs.
*   **Parameters:**
    *   `streak_length`: Integer (the active streak count)
    *   `time_of_day`: String (e.g., "morning", "afternoon", "evening")

### 1.3 `streak_freeze_used`
Triggered when a freeze token is toggled.
*   **Parameters:**
    *   `remaining_freezes`: Integer (freeze token count left)

## 2. Standard Telemetry & Crashlytics
*   `app_exception`: Automatically logged by Firebase Crashlytics on uncaught crashes.
*   `screen_view`: Mapped to primary destinations (`dashboard_screen`, `calendar_analytics`, `settings_screen`).
