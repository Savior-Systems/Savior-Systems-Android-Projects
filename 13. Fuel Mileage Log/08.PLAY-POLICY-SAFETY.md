# 08. Play Store Policy & Data Safety
```
This document outlines Google Play Console policy compliance guidelines and the data safety profile for **Fuel Mileage Log**.
```
---
```
## 1. Requested Permissions
To maximize user trust and prevent automated app flags during review, the app requests **zero permissions**.
```
No location, contact, phone, storage (using internal sandboxed DB files), or hardware access permissions are declared.
```
---
```
## 2. Google Play Data Safety Declarations
```
### A. Data Collection
*   **Data Collected**: None.
*   The application does not transmit any user data (e.g., fuel records, cost settings) to external servers. Everything is stored locally in the Room SQLite database.
```
### B. Data Sharing
*   **Data Shared**: None.
*   No personal data, device identifiers, or mileage records are shared with third-party networks or service providers.
```
### C. Data Security Measures
*   **Encryption**: No core user database data ever leaves the device.
*   **Data Deletion**: Users can delete their data at any time by tapping "Reset Data" in Settings.
```
