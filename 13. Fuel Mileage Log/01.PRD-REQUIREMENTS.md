# 01. Product Requirement Document (PRD)
```
## 1. Vision & Strategy
*   **App Name**: Fuel Mileage Log
*   **Category**: Auto & Vehicles / Finance
*   **Core Objective**: Build a fast, offline-first mileage and fuel tracker. It directly targets search intent for the keywords: `fuel log`, `gas mileage calculator`, `car mileage tracker`, and `fuel efficiency logs`.
*   **Value Proposition**: No internet required, no cloud accounts, instantaneous fuel logging, and elegant fuel economy analytics.
```
---
```
## 2. User Personas
### Persona: Marcus (Gig Delivery Driver)
*   **Needs**: Fast, on-the-go logging of fuel purchases, odometer readings, and costs, with instant calculation of fuel consumption statistics.
*   **Pain Points**: Clunky cloud apps that require network connectivity to save fuel receipts, loading slowly in remote areas.
*   **How the App Solves it**: The application stores all fuel history locally in a Room database, running 100% offline. Marcus opens the app, types the current odometer, fuel quantity, and price, and taps "Log".
```
---
```
## 3. Core Feature List
1.  **Instant Fuel Entry**:
    *   One-tap add dialog/screen.
    *   Fields: Odometer (km/miles), Fuel Volume (liters/gallons), Total Cost, and Date.
2.  **Interactive Mileage Dashboard**:
    *   Key Stats: Average fuel consumption (L/100km or MPG), Total Cost, Cost per km/mile, and Distance travelled since last fill-up.
3.  **Fuel History Log**:
    *   Scrollable list of past logs showing detailed fill-up history.
4.  **Local Backup & Settings**:
    *   Unit selector (Metric/Imperial).
    *   Export/Import data in JSON or CSV formats.
    *   Reset data action.
```
---
```
## 4. Monetization & Ad Zones
*   **Zero-Ad Zones**:
    *   The entry screen and calculation form are strictly zero-ad to prevent entry latency.
*   **Monetization**:
    *   Anchored adaptive banner at the bottom of the dashboard.
    *   Interstitial ad on exiting Settings or after completing a CSV export, subject to a **180s cooldown**.
```
