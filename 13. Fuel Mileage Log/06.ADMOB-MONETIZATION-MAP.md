# 06. AdMob Monetization Map

This document outlines the AdMob monetization integrations, placement parameters, and zero-ad configurations for **Fuel Mileage Log**.

---

## 1. Placements
*   **Adaptive Banner**: Positioned at the bottom of the dashboard screen.
*   **Interstitial Ad**: Triggered only upon exporting fuel log data as CSV/JSON or clearing histories in Settings.
*   **Zero-Ad Focus Zones**: The main fuel logging form modal/dialog must remain strictly ad-free to keep the logging workflow fast and efficient.

---

## 2. 180s Cooldown Guard
A global cooldown is enforced to prevent interstitial ad spam. The `AdMobHelper` class tracks the last timestamp an ad was shown and blocks new requests if less than 180 seconds have elapsed.

```kotlin
package com.saviorsystems.fuelmileagelog.utils

object AdMobHelper {
    private var lastAdShowTime: Long = 0L
    private const val COOLDOWN_MS = 180_000L // 3 minutes

    fun canShowInterstitial(): Boolean {
        return (System.currentTimeMillis() - lastAdShowTime) >= COOLDOWN_MS
    }

    fun markAdShown() {
        lastAdShowTime = System.currentTimeMillis()
    }
}
```
