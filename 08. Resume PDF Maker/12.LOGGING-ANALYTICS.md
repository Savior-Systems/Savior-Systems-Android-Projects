# 12. Logging & Analytics

## 1. Firebase Analytics Schema (Non-PII Event Telemetry)
To measure user engagement and funnels without collecting personal resume data, all logged parameters use generic tracking keys.

### A. Core Action Events
* **`resume_create`:**
  * **Description:** Logged when a user clicks "Create New Resume".
  * **Parameters:** `template_preset` (String)
* **`pdf_export_start`:**
  * **Description:** Logged when the user clicks the "Export PDF" button.
  * **Parameters:** `template_name` (String), `margin_style` (String), `page_count_estimate` (Int)
* **`pdf_export_success`:**
  * **Description:** Logged when the PDF document is successfully saved to local storage.
  * **Parameters:** `duration_ms` (Int), `file_size_kb` (Int), `page_count` (Int)
* **`pdf_export_failed`:**
  * **Description:** Logged if the PDF generator fails due to layout overflows or canvas errors.
  * **Parameters:** `error_message` (String), `template_name` (String)

### B. Monetization & Integration Events
* **`rewarded_ad_request`:** Logged when the watch-ad dialog is shown.
* **`rewarded_ad_complete`:** Logged when the user successfully finishes watching the video ad and receives the 24h premium customization token.
* **`json_backup_export`:** Logged when a JSON backup file is successfully written.
* **`json_backup_import`:** Logged when a JSON profile is successfully loaded.
* **`intent_open_external`:** Logged when the user clicks the Share/Open PDF chooser to open the file in Canva or another external reader.

## 2. Firebase Crashlytics & Non-Fatal Exception Logging
PDF drawing commands and file system operations can fail on specific Android device distributions due to native font rendering differences or storage restrictions.

### Logging Exceptions:
* **`PdfDocument` Writing Failures:** If `PdfDocument.writeTo()` throws an `IOException`, log the exception as a non-fatal error:
  ```kotlin
  FirebaseCrashlytics.getInstance().recordException(e)
  ```
* **JSON Parsing Errors:** If imported JSON backups are corrupt or fail validation checks, log a custom key `json_import_error` alongside the error message.
* **Page Budget Overflows:** If `PageBudgetingEngine` detects an infinite layout loop (e.g. an element too tall for a single page), throw and capture the error to prevent app locks.
