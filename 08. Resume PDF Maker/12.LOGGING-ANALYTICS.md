## 1. Firebase Custom Events
`
We register 4 custom events to evaluate app engagement:
`
### Event A: `template_selected`
*   **Trigger**: User taps a template option.
*   **Parameters**:
    *   `template_id` (string: "classic", "teal", "navy", "minimalist")
`
### Event B: `pdf_compiled_success`
*   **Trigger**: PDF layout rendering resolves and print manager starts.
*   **Parameters**:
    *   `duration_ms` (integer: render time)
`
### Event C: `cv_section_added`
*   **Trigger**: User adds an additional row to lists (e.g. dynamic Work details).
*   **Parameters**:
    *   `section_name` (string: "education", "experience", "projects", "languages")
`
### Event D: `editor_action`
*   **Trigger**: Clicks to clear profile data.
*   **Parameters**:
    *   `action_type` (string: "reset_all")
`

## 2. Strict Privacy Guardrails ("NEVER Log" Rules)
`
To remain 100% compliant with privacy regulations, we enforce the following restrictions:
*   🚫 **NEVER log CV Input values**: Personal details (names, email addresses, phone numbers) must never be captured or logged in analytics parameters.
*   🚫 **NEVER log Job/Company descriptors**: Job names, company details, or university names are forbidden in parameter logs.
*   🚫 **NEVER log Resume files**: Document files or base64 streams must never leave the local cache or print spooler pipeline.
`
