# 01. Product Requirements Document (PRD)

## 1. Project Context & Vision
**App Name:** Resume PDF Maker  
**Category:** Productivity / Tools  
**Core Vision:** A highly robust, 100% offline-first application that enables users to create, customize, and export professional, ATS-friendly resumes in minutes. The application compiles structured CV data into vector-based, selectable-text PDFs that are fully editable, parsed correctly by ATS crawlers, and compatible with external productivity tools (like Canva, Adobe Acrobat, and Google Docs).

---

## 2. Target Audience
* **Job Seekers & Students:** Creating clean CVs matching corporate standards.
* **Professionals:** Requiring selectable, high-fidelity ATS-compliant formats.
* **Freelancers/Contractors:** Managing multiple target CV variants and custom profiles.

---

## 3. Scope of MVP (Minimum Viable Product)
### In-Scope Features:
1. **Multi-Profile CV System:** Create and manage multiple distinct resumes (e.g. "Software Dev", "Product Manager") from a unified dashboard.
2. **Dynamic Data Entry Forms:** Section-by-section input forms with input validation:
   * **Personal Info & Summary:** Contact links, LinkedIn/GitHub, professional summaries.
   * **Work Experience:** Companies, roles, durations, and bullet points.
   * **Education:** Institutions, degrees, locations, and graduation years.
   * **Core Skills & Languages:** Grouped skills (e.g., Languages, Frameworks) with proficiency indicators.
   * **Projects & Certifications:** Structured grids with detail lists.
3. **Template Personalization Engine:**
   * **Visual Customization:** Adjust margins (Compact, Normal, Wide), paragraph spacing, and line heights.
   * **Font Pairings:** Choose font families (Serif for Harvard style, Sans-serif for Modern/ATS).
   * **Branding Palettes:** Dynamic selection of corporate color themes (Classic Navy, Emerald Corporate, Onyx Modern).
4. **Editable & Selectable Vector PDF Generator:**
   * Generates standard A4 PDF files using native Android `PdfDocument` canvas drawing.
   * **Selectable Text Elements:** All text must be drawn as native font glyphs rather than flat images. This ensures the output is 100% searchable, ATS-friendly, and fully editable in PDF applications (like Adobe Acrobat or Canva).
5. **Standard Productivity Integrations:**
   * **Direct Open/View Intent:** On export completion, trigger an Android `Intent.ACTION_VIEW` with MIME type `application/pdf` to let the user open and edit the document in external applications (e.g., Canva, Drive, Adobe Reader).
   * **Direct Share Intent:** Trigger `Intent.ACTION_SEND` to share files directly.
6. **Local Backup & Restore:**
   * Export raw CV profile data as a formatted JSON file (`.json`) to local storage.
   * Import previously exported JSON profiles to restore data offline.
7. **Real-time Canvas Preview:**
   * An in-app print preview that renders the A4 layout page-by-page using Compose `Canvas` to display margins, font scaling, and page boundaries without blocking disk writes.

---

## 4. Rookie Developer Guidelines
* **Zero-Permission File Exports:** Use Scoped Storage and the Storage Access Framework (SAF) to save PDFs into the public `/Downloads` directory without requesting broad, dangerous storage read/write permissions.
* **ATS Compatibility Rule:** **NEVER** rasterize the canvas into a bitmap before writing to the PDF. Bitmaps make the text unselectable. Use `Canvas.drawText()` and native font paths so standard parsers can extract the text.
* **Print Page Boundaries:** Standard A4 dimensions are exactly `595 x 842` PostScript points. Implement a height-budgeting engine to track vertical drawing limits and create clean multi-page splits without splitting a line of text in half.
