# 01. Product Requirement Document (PRD) — Resume PDF Maker
```
## 1. Vision & Strategy
*   **App Name**: Resume PDF Maker
*   **Category**: Productivity / Utilities
*   **Core Objective**: Build a highly performant, 100% offline-first application that allows users to create, format, preview, and generate professional PDF resumes and CVs using dynamic localized templates.
*   **Target Market**: Global job seekers, students, and professionals looking for a secure, offline, and free CV building application without cloud subscriptions.
```
---
```
## 2. Target Personas
```
### Persona A: Rahul Sharma (22, Mumbai, India)
*   **Role**: Fresh Engineering Graduate.
*   **Pain Point**: Needs to draft different versions of his resume for IT vs. Core jobs, but standard builders require expensive subscriptions or internet connections to export PDFs.
*   **Usage**: Uses Resume PDF Maker to save multiple offline profiles and generate clean, professional single-page PDFs instantly.
```
### Persona B: Maria Lindstrom (38, Berlin, Germany)
*   **Role**: Freelance Marketing Consultant.
*   **Pain Point**: Values data privacy and does not want to upload sensitive personal details (address, phone number, employment history) to online databases.
*   **Usage**: Appreciates the 100% offline database, saving her career history locally and exporting templates on the fly.
```
---
```
## 3. Core Feature List & User Stories
```
### Feature 1: Resume Profile Builder (Form Sections)
*   **Requirement**: Dedicated input forms for Personal Info, Objective, Education, Experience, Projects, Skills, and Languages. Support adding/removing custom list entries.
*   **User Story**: *As Rahul, I want to edit my educational details and projects separately so that my database of career history remains structured.*
```
### Feature 2: Offline Templates Selector & Style Customizer
*   **Requirement**: Choose from 4 professional templates (Modern Classic, Creative Teal, Executive Navy, Minimalist). Adjust font size and line height dynamically.
*   **User Story**: *As Maria, I want to switch between different layout templates with a single click so that I can tailor the visual style for different clients.*
```
### Feature 3: Web-to-PDF Print Adapter Exporter
*   **Requirement**: Render structured database fields into a highly styled HTML resume template within a headless `WebView`, and trigger the Android system printer to export as a high-fidelity PDF document.
*   **User Story**: *As Rahul, I want to export my resume directly to a PDF file on my phone storage so that I can submit it to recruiters immediately.*
```
---
```
## 4. Key Performance Indicators (KPIs)
*   **Cold Start Time**: < 450ms.
*   **PDF Compile Speed**: < 1.5 seconds from form click to system print spooler.
*   **Offline Security**: 100% offline. Zero network traffic containing personal details.
```
