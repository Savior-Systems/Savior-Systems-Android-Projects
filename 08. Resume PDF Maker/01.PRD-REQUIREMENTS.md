# 01. Product Requirements Document (PRD)

## 1. Project Context & Vision
**App Name:** Resume PDF Maker  
**Category:** Productivity / Tools  
**Core Vision:** An offline-first, highly robust application that allows users to create professional, ATS-friendly resumes in minutes. The app must dynamically compile user input (Education, Experience, Skills) into multiple professional PDF templates.

## 2. Target Audience
- Students looking for their first job.
- Professionals needing quick, ATS-compliant formats.
- Freelancers maintaining multiple versions of their CVs.

## 3. Scope of MVP (Minimum Viable Product)
### In-Scope Features:
1. **Multi-Profile System:** Users can create and manage multiple distinct resumes.
2. **Dynamic Forms:** Stepper-based or Section-based input forms for:
   - Personal Details & Objective
   - Education History
   - Work Experience
   - Skills & Languages
   - Projects & Certifications
3. **Template Engine:** At least 3 robust PDF templates:
   - **Minimalist/ATS:** Standard black and white, text-heavy.
   - **Modern:** Two-column layout with a custom accent color.
   - **Harvard Style:** Academic, deeply structured formatting.
4. **Export Engine:** Native PDF generation using Android `PdfDocument` or equivalent.
5. **Preview Mode:** In-app PDF viewer or scaled Canvas preview before exporting.

### Out-of-Scope (Future Updates):
- Cover letter generator.
- AI (ChatGPT) integration for writing bullet points.
- Cloud syncing (Google Drive).

## 4. Rookie Developer Guidelines
- **Data Integrity is King:** A resume with a missing phone number or corrupted dates is useless. Ensure Room Database transactions are flawless.
- **Relational Databases:** This is not a simple table. A Resume has a 1-to-N relationship with Experiences. Master Room Relations (`@Relation`).
- **PDF Sizing:** A4 size is standard (595 x 842 PostScript points). Map UI pixels to PDF points carefully.
