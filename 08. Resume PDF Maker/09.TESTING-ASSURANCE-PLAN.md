## 2. Manual QA Matrix (Edge Cases)
```
| QA Case | Action | Expected Behavior |
| :--- | :--- | :--- |
| **Special Character Names** | Input Unicode/non-Latin characters (e.g. `Müller`, `আহমেদ`). | WebView renders correctly, exporting characters to PDF without encoding errors. |
| **Massive Descriptions** | Add a 2000-character description in the Job Experience field. | CSS page-breaker triggers correctly; content flows gracefully to page 2 instead of cutting off. |
| **Partial Profiling** | Fill in only "Name" and leave "Experience" and "Skills" empty. | PDF template compiles successfully, omitting empty sections without rendering blank borders. |
| **No-Permissions Print** | Trigger PDF export with system notifications disabled. | PDF creation proceeds normally via system print framework. |
```
---
```
## 3. Performance Benchmarks
*   **App Boot Target**: `< 450ms` cold boot to interactive editor fields.
*   **Memory Footprint**: App heap allocation must not exceed `80MB` during print rendering.
*   **Page Rendering Delay**: Headless WebView must complete DOM compiling and handoff to the print manager in `< 1.2s`.
