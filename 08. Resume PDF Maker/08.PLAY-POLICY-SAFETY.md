# 08. Play Policy & Data Safety — Resume PDF Maker
`
Maintaining user trust by enforcing local sandboxes and stripping invasive permissions.
`
---
`
## 1. System Permissions Audit
`
### Required Permissions
*   No dangerous runtime permissions are requested.
*   Uses native system printing capabilities via standard Android `PrintManager`, requiring zero storage permissions on Android 10+ (API 29+).
`
### Stripped / Unused Permissions (🚫 Zero Tolerance)
*   🚫 `android.permission.READ_EXTERNAL_STORAGE` (Not required for system PDF print spooler)
*   🚫 `android.permission.WRITE_EXTERNAL_STORAGE` (Not required on modern SDKs)
*   🚫 `android.permission.ACCESS_FINE_LOCATION` (Zero location logging)
*   🚫 `android.permission.READ_CONTACTS` (User info is typed manually)
`

`
## 2. Play Console Data Safety Questionnaire
`
Since all personal data is kept inside the local Room database, the Data Safety questionnaire will declare the following:
`
| Section Question | Answer Status | Rationale |
| :--- | :--- | :--- |
| **Does your app collect or share user data?** | **No** | All user inputs (names, jobs, phone numbers) remain 100% locally on the device and are never shared. |
| **Is all user data encrypted in transit?** | **N/A** | No data is transmitted. |
| **Can users delete their account/data?** | **Yes** | Users can wipe the entire profile repository instantly via Settings -> Reset App. |
`
