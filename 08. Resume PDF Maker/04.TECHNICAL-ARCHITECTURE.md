# 04. Technical Architecture Blueprint

## 1. Architecture Choice: MVVM (Model-View-ViewModel)
Strict separation of concerns. The complexity of PDF generation must not leak into the UI components.

## 2. Recommended Tech Stack
- **Language:** Kotlin
- **UI:** Jetpack Compose (Material 3)
- **Local Database:** Room Database (Relational)
- **Dependency Injection:** Hilt
- **Asynchrony:** Kotlin Coroutines & Flow
- **PDF Engine:** Android `PdfDocument` (Native) or a lightweight wrapper.

## 4. PDF Generation Architecture
- Create an interface `PdfGenerator { fun generate(data: ResumeWithDetails, uri: Uri) }`.
- Implement different classes (`ModernTemplate`, `HarvardTemplate`) that implement this interface. This ensures the Open/Closed Principle.
