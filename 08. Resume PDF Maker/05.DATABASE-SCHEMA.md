# 05. Database Schema & Storage

## 1. Relational Database Design
A Resume is a complex aggregate root. We must use Room Relations.

### Table: `resumes` (The Parent)
| Column        | Type   | Description |
|---------------|--------|-------------|
| `resumeId`    | Long   | PK, Auto-Inc|
| `profileName` | String | e.g. "Software Dev Resume" |
| `fullName`    | String | Personal Detail |
| `email`       | String | Personal Detail |
| `phone`       | String | Personal Detail |
| `templateId`  | Int    | Which template to use |

### Table: `educations` (Child)
| Column        | Type   | Description |
|---------------|--------|-------------|
| `eduId`       | Long   | PK, Auto-Inc|
| `resumeOwnerId`| Long  | FK -> resumes(resumeId) CASCADE |
| `institution` | String | School Name |
| `degree`      | String | e.g. "B.Sc CS" |
| `startYear`   | String | |
| `endYear`     | String | |

### Table: `experiences` (Child)
*Columns similar to education (company, role, start, end, description).*

### Table: `skills` (Child)
*Columns: skillId, resumeOwnerId, skillName.*

## 2. Room Relation Mapping (Crucial)
To fetch a resume and all its data at once, create a POJO:
```kotlin
data class ResumeWithDetails(
    @Embedded val resume: ResumeEntity,
    
    @Relation(parentColumn = "resumeId", entityColumn = "resumeOwnerId")
    val educations: List<EducationEntity>,
    
    @Relation(parentColumn = "resumeId", entityColumn = "resumeOwnerId")
    val experiences: List<ExperienceEntity>,
    
    @Relation(parentColumn = "resumeId", entityColumn = "resumeOwnerId")
    val skills: List<SkillEntity>
)
```

## 3. DAO Queries
```kotlin
@Transaction
@Query("SELECT * FROM resumes WHERE resumeId = :id")
fun getResumeWithDetails(id: Long): Flow<ResumeWithDetails>
```
