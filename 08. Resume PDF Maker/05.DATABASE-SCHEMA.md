# 05. Database Schema & Storage

## 1. Relational Database Design
A Resume is a complex aggregate root representing a portfolio configuration. The database handles parent-child relations with cascade deletion rules to prevent orphaned data.

```
  ┌──────────────┐
  │   resumes    │◄────────────────────────────────┐
  └──────┬───────┘                                 │
         │ (1-to-1)                                │
         ▼                                         │
  ┌──────────────┐                                 │ (1-to-Many)
  │temp_configs  │                                 │
  └──────────────┘                                 │
                                                   │
     ┌───────────────┬───────────────┬─────────────┼──────────────┐
     ▼               ▼               ▼             ▼              ▼
┌──────────┐   ┌──────────┐    ┌──────────┐  ┌──────────┐   ┌──────────┐
│educations│   │experience│    │  skills  │  │ projects │   │certificat│
└──────────┘   └──────────┘    └──────────┘  └──────────┘   └──────────┘
```

## 2. Table Specifications

### Table: `resumes` (The Parent)
| Column | Type | Constraints | Description |
|:---|:---|:---|:---|
| `resumeId` | Long | PK, Auto-Inc | Unique identifier |
| `profileName` | String | NOT NULL | e.g. "Google Software Engineer" |
| `fullName` | String | NOT NULL | Personal Detail |
| `email` | String | NOT NULL | Personal Detail |
| `phone` | String | NOT NULL | Personal Detail |
| `website` | String | | GitHub / LinkedIn / Portfolio URL |

### Table: `template_configs` (Template Preferences)
| Column | Type | Constraints | Description |
|:---|:---|:---|:---|
| `configId` | Long | PK, Auto-Inc | Unique identifier |
| `resumeOwnerId`| Long | FK -> `resumes(resumeId)` | CASCADE on delete |
| `templateName` | String | NOT NULL | e.g., "Classic Navy", "Onyx Modern" |
| `marginSize` | String | NOT NULL | e.g., "Compact", "Normal", "Wide" |
| `fontPairing` | String | NOT NULL | e.g., "Modern Sans", "Classic Serif" |

### Table: `experiences` (Child)
| Column | Type | Constraints | Description |
|:---|:---|:---|:---|
| `expId` | Long | PK, Auto-Inc | Unique identifier |
| `resumeOwnerId`| Long | FK -> `resumes(resumeId)` | CASCADE on delete |
| `company` | String | NOT NULL | Company name |
| `role` | String | NOT NULL | Job Title |
| `duration` | String | NOT NULL | e.g., "Jan 2024 - Present" |
| `description` | String | NOT NULL | Bullet points joined by newline (`\n`) |

### Table: `projects` (Child)
| Column | Type | Constraints | Description |
|:---|:---|:---|:---|
| `projectId` | Long | PK, Auto-Inc | Unique identifier |
| `resumeOwnerId`| Long | FK -> `resumes(resumeId)` | CASCADE on delete |
| `projectName` | String | NOT NULL | Project Name |
| `techStack` | String | NOT NULL | e.g., "Kotlin, Jetpack Compose, Room" |
| `description` | String | NOT NULL | Project description bullet points |

### Table: `certifications` (Child)
| Column | Type | Constraints | Description |
|:---|:---|:---|:---|
| `certId` | Long | PK, Auto-Inc | Unique identifier |
| `resumeOwnerId`| Long | FK -> `resumes(resumeId)` | CASCADE on delete |
| `certName` | String | NOT NULL | e.g., "Associate Android Developer" |
| `issuer` | String | NOT NULL | e.g., "Google" |
| `year` | String | NOT NULL | e.g., "2026" |

* **Note:** `educations` (columns: `eduId`, `institution`, `degree`, `duration`, `gpa`) and `skills` (columns: `skillId`, `skillName`, `skillCategory`) follow identical relation mapping rules.

## 3. Room Relation Mapping (POJO Aggregate)
To load a complete resume profile with all its related records in a single database transaction:

```kotlin
package com.saviorsystems.resumemaker.data.local.entity.relations

import androidx.room.Embedded
import androidx.room.Relation
import com.saviorsystems.resumemaker.data.local.entity.*

data class ResumeWithDetails(
    @Embedded val resume: ResumeEntity,

    @Relation(parentColumn = "resumeId", entityColumn = "resumeOwnerId")
    val templateConfig: TemplateConfigEntity?,

    @Relation(parentColumn = "resumeId", entityColumn = "resumeOwnerId")
    val educations: List<EducationEntity>,

    @Relation(parentColumn = "resumeId", entityColumn = "resumeOwnerId")
    val experiences: List<ExperienceEntity>,

    @Relation(parentColumn = "resumeId", entityColumn = "resumeOwnerId")
    val projects: List<ProjectEntity>,

    @Relation(parentColumn = "resumeId", entityColumn = "resumeOwnerId")
    val certifications: List<CertificationEntity>,

    @Relation(parentColumn = "resumeId", entityColumn = "resumeOwnerId")
    val skills: List<SkillEntity>
)
```
