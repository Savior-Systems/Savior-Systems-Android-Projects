# 13. Step-by-Step Implementation Guide

Follow these development phases sequentially to build, test, and release **Resume PDF Maker**:

---

## 🛠️ Phase 1: Database Schema & local JSON models (Days 1-2)
* **Goal:** Set up relational tables and JSON serialization models.
* **Tasks:**
  * [ ] Create entity classes: `ResumeEntity`, `EducationEntity`, `ExperienceEntity`, `ProjectEntity`, `CertificationEntity`, `SkillEntity`, and `TemplateConfigEntity`.
  * [ ] Add `ForeignKey` annotations with `onDelete = ForeignKey.CASCADE` linking to parent `resumeId`.
  * [ ] Create Room POJO `ResumeWithDetails` embedding all relations.
  * [ ] Write JUnit database tests to verify cascading deletions (deleting a resume must delete all associated child rows).
  * [ ] Define `ResumeBackup` data model annotated with `Kotlinx.Serialization` or `Gson` for offline JSON import/export.

---

## ⚙️ Phase 2: ViewModels, JSON Share & SAF File Handlers (Day 3)
* **Goal:** Implement state management and offline backup storage.
* **Tasks:**
  * [ ] Create `EditorViewModel` maintaining a `StateFlow<ResumeWithDetails>` backed by `SavedStateHandle`.
  * [ ] Build local JSON serializer helper `JsonBackupManager`.
  * [ ] Implement Storage Access Framework (SAF) `CreateDocument` and `OpenDocument` launchers in `MainActivity`.
  * [ ] Connect JSON export and import actions to UI triggers, verifying successful backup restore.

---

## 🎨 Phase 3: Forms, Preview Canvas & Customization Panel (Days 4-5)
* **Goal:** Implement the data entry forms and real-time print preview.
* **Tasks:**
  * [ ] Build `DashboardScreen` displaying saved resumes, import buttons, and native ads.
  * [ ] Build `EditorHubScreen` using expandable accordions for section entry.
  * [ ] Implement form inputs with field validation (phone, email, URLs).
  * [ ] Build `PreviewScreen` drawing layout outlines on Compose `Canvas` using print margins and font scaling parameters.
  * [ ] Create `StyleConfigPanel` containing margin sliders, font dropdowns, and corporate color presets.

---

## 📐 Phase 4: PDF Layout Templates & PageBudgetingEngine (Days 6-8)
* **Goal:** Implement the vector-text PDF rendering engine and page partition algorithm.
* **Tasks:**
  * [ ] Define `PrintableSection` interface and implement it on database item blocks.
  * [ ] Write the `PageBudgetingEngine` class calculating text height and page boundaries.
  * [ ] Build templates: `ClassicNavyTemplate`, `EmeraldCorporateTemplate`, and `OnyxModernTemplate` drawing selectable vector text.
  * [ ] Test page boundaries: verify that overflow text dynamically splits across pages.

---

## 🚀 Phase 5: AdMob Premium Unlocks & Sharing Intents (Day 9)
* **Goal:** Integrate rewarded video ads and system intents.
* **Tasks:**
  * [ ] Integrate AdMob SDK and configure `ResumeAdManager` with 180s interstitial cooldown.
  * [ ] Implement Rewarded Video Ads unlocking premium layout styles for 24 hours.
  * [ ] Set up `Intent.ACTION_VIEW` and `Intent.ACTION_SEND` choosers on PDF export completion.
  * [ ] Verify opening the exported PDF inside Canva, Google Drive, and Gmail.
