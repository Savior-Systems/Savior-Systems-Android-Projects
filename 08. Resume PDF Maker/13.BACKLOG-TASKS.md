# 13. Step-by-Step Implementation Guide (Rookie Roadmap)

**Dear Rookie Developer,**
Resume PDF Maker is a complex app because of the PDF generation and relational database. Follow these steps exactly. Do not build the UI until the DB is perfect.

### Phase 1: Foundation & Database (Days 1-2)
1. Setup Project, Hilt, Room, and Navigation.
2. Create `ResumeEntity`, `EducationEntity`, `ExperienceEntity`, and `SkillEntity`. Set up the `ForeignKey` constraints with `CASCADE` delete.
3. Create the `ResumeWithDetails` data class.
4. Create `ResumeDao` and write tests or use Database Inspector to manually verify that inserting a child row properly links to the parent resume.

### Phase 2: ViewModels & State (Day 3)
1. Create `EditorViewModel`.
2. Write functions: `savePersonalDetails()`, `addEducation()`, `deleteEducation()`.
3. Ensure the ViewModel emits a complete `ResumeWithDetails` state flow.

### Phase 3: Compose Forms UI (Days 4-5)
1. Build `DashboardScreen` (List of Resumes).
2. Build `EditorHubScreen` (Grid of sections).
3. Build the Forms. Use `OutlinedTextField`. Ensure keyboard types are correct (e.g., `KeyboardType.Email` for email field).

### Phase 4: PDF Engine (Days 6-8) [HARDEST PHASE]
1. Start simple. Create a `MinimalistPdfGenerator` class.
2. Use Android `PdfDocument.PageInfo.Builder(595, 842, 1).create()`.
3. Use `Canvas.drawText()` to render the `ResumeWithDetails` data onto the page.
4. Calculate Y-offsets. E.g., draw Name at Y=50. Draw Email at Y=80. 
5. Implement `Intent.ACTION_CREATE_DOCUMENT` to let the user save the `PdfDocument` to their storage.

### Phase 5: Polish & Ads (Day 9)
1. Add AdMob Banners to the Dashboard.
2. Add the Interstitial Ad trigger after PDF export.
3. Test edge cases (extremely long names, null fields). Build the Release APK.
