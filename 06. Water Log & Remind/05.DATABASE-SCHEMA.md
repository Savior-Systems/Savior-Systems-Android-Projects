# 05. Database Schema & DataStore — Water Log & Remind

The persistence layer relies on Room for historical tracking (crucial for charting and trends) and DataStore for real-time user state (goal, streaks, settings).

---

## 1. Room Database Schema

### Table: `intake_logs`
Stores every individual drink event.

| Column | Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `id` | Long | **PK**, AutoGenerate | Unique ID for the log entry. |
| `timestamp` | Long | Not Null | Exact Unix timestamp of the drink. |
| `date_id` | String | Not Null, **Indexed**| Format: `YYYY-MM-DD`. Used for fast grouping/aggregation. |
| `amount_ml` | Int | Not Null | Volume of the drink in milliliters. |
| `beverage_type`| String | Not Null | Enum: `WATER`, `COFFEE`, `TEA`, `JUICE`, `SODA`. |
| `hydration_factor`| Float | Not Null | Multiplier (e.g., Water = 1.0, Coffee = 0.8). Effective hydration = `amount_ml * hydration_factor`. |

### Table: `daily_summaries`
A rollup table updated daily by the `MidnightResetWorker` to prevent querying thousands of individual logs when rendering monthly charts.

| Column | Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `date_id` | String | **PK**, Not Null | Format: `YYYY-MM-DD`. |
| `total_ml` | Int | Not Null | Sum of all effective hydration for the day. |
| `goal_ml` | Int | Not Null | The user's goal on that specific day. |
| `goal_met` | Boolean | Not Null | True if `total_ml` >= `goal_ml`. |

## 2. Room DAO Queries

```kotlin
@Dao
interface IntakeDao {

    // Get today's total progress (Used by DashboardViewModel and Widget)
    @Query("SELECT SUM(amount_ml * hydration_factor) FROM intake_logs WHERE date_id = :todayId")
    fun getTodaysTotalFlow(todayId: String): Flow<Int?>

    // Get detailed log list for today's timeline
    @Query("SELECT * FROM intake_logs WHERE date_id = :todayId ORDER BY timestamp DESC")
    fun getTodaysLogsFlow(todayId: String): Flow<List<IntakeLogEntity>>

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insertLog(log: IntakeLogEntity)

    @Delete
    suspend fun deleteLog(log: IntakeLogEntity)
    
    // --- Summary Queries (For Charts) ---
    @Query("SELECT * FROM daily_summaries WHERE date_id BETWEEN :startDate AND :endDate ORDER BY date_id ASC")
    suspend fun getSummariesForRange(startDate: String, endDate: String): List<DailySummaryEntity>
}
```

## 3. DataStore Preferences (Profile & State)

All user configurations and real-time streaks are stored in `DataStore<Preferences>`.

| Preference Key | Type | Default Value | Description |
| :--- | :--- | :--- | :--- |
| `pref_weight_kg` | Float | `70.0` | User's saved weight. |
| `pref_gender` | String | `"UNSPECIFIED"` | "MALE", "FEMALE", "UNSPECIFIED". |
| `pref_activity` | String | `"MODERATE"` | "SEDENTARY", "MODERATE", "ACTIVE". |
| `pref_daily_goal` | Int | `2000` | Current daily goal in ml. |
| `pref_unit` | String | `"ML"` | Display unit: "ML" or "OZ". Database always stores ML. |
| `pref_current_streak`| Int | `0` | Consecutive days the goal was met. |
| `pref_best_streak` | Int | `0` | All-time highest streak. |

### Alarm/Reminder Configurations
| Preference Key | Type | Default Value | Description |
| :--- | :--- | :--- | :--- |
| `reminders_enabled`| Boolean | `true` | Master toggle for notifications. |
| `wake_time` | String | `"07:00"` | HH:MM format. Start of active hours. |
| `sleep_time` | String | `"22:30"` | HH:MM format. End of active hours. |
| `reminder_interval`| Int | `90` | Minutes between reminders (e.g., 60, 90, 120). |
| `smart_reminders` | Boolean | `true` | Adjusts reminder frequency if user is already ahead of schedule. |

## 4. Midnight Rollover & Streak Logic
The `MidnightResetWorker` executes at 00:00:
1. Queries `IntakeDao` for yesterday's sum.
2. Reads `pref_daily_goal` from DataStore.
3. Writes a new record to `daily_summaries`.
4. If yesterday's sum >= goal, increment `pref_current_streak`. If the new streak > `pref_best_streak`, update best.
5. If yesterday's sum < goal, reset `pref_current_streak` to 0.
