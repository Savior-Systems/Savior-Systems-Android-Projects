# 12. Logging & Analytics — Water Log & Remind

This document defines the Firebase Analytics telemetry schema, custom event parameters, and Crashlytics configuration. We prioritize privacy and collect **zero personally identifiable health information (PII)**.

---

## 1. Firebase Analytics Configuration

### 1.1 SDK Integration
*   **BOM Version**: Firebase BOM 33+ (latest stable).
*   **Dependencies**: `firebase-analytics-ktx`, `firebase-crashlytics-ktx`.
*   **Consent**: Analytics collection starts on first app launch for non-EU users. EU users must pass a UMP consent gate.

### 1.2 Automatic Events
Firebase automatically logs: `first_open`, `session_start`, `app_update`, `os_update`.

---

## 2. Custom Firebase Events

### 2.1 `onboarding_completed`
Triggered when the user finishes the initial setup and sees their calculated goal.
*   **Parameters:**
    *   `calculated_goal_bucket`: String (e.g., `"<1500"`, `"1500-2000"`, `"2000-3000"`, `">3000"`)
*   **Purpose**: Understand what typical hydration goals are being assigned.

### 2.2 `water_logged`
Triggered whenever a user adds a drink.
*   **Parameters:**
    *   `source`: String (`"app_dashboard"`, `"home_widget"`, `"notification_action"`)
    *   `beverage_type`: String (`"water"`, `"coffee"`, `"tea"`, etc.)
    *   `amount_bucket`: String (`"<100"`, `"100-250"`, `"250-500"`, `">500"`)
*   **Purpose**: Crucial metric to determine the adoption rate of the Jetpack Glance widget vs. in-app logging.

### 2.3 `goal_achieved`
Triggered when the daily progress hits 100%.
*   **Parameters:**
    *   `time_of_day`: String (`"morning"`, `"afternoon"`, `"evening"`, `"night"`)
*   **Purpose**: Track user success rates and identify when people typically finish hydrating.

### 2.4 `streak_milestone`
Triggered during the Midnight Reset if the user hits a specific streak threshold.
*   **Parameters:**
    *   `streak_days`: Integer (`3`, `7`, `14`, `30`, `100`)
*   **Purpose**: Measure long-term retention and gamification effectiveness.

### 2.5 `reminder_settings_changed`
Triggered when a user alters their notification schedule.
*   **Parameters:**
    *   `interval_minutes`: Integer (`30`, `60`, `90`, `120`)
    *   `smart_reminders_enabled`: String (`"true"`, `"false"`)
*   **Purpose**: Understand preferred reminder cadences.

## 5. Crashlytics Configuration

### 5.1 Crash Reporting
*   **Automatic**: All uncaught exceptions are captured.
*   **Non-Fatal Logging**: Use `Firebase.crashlytics.recordException(e)` for caught exceptions in `Room` DB migrations or `Widget` update failures.

### 5.2 Custom Keys for Crash Context
```kotlin
Firebase.crashlytics.setCustomKey("widget_active", isWidgetInstalled)
Firebase.crashlytics.setCustomKey("db_size_logs", totalLogsCount)
Firebase.crashlytics.setCustomKey("can_schedule_exact_alarms", alarmManager.canScheduleExactAlarms())
```

### 5.3 Breadcrumbs
Log non-fatal breadcrumbs for debugging crash context:
```kotlin
Firebase.crashlytics.log("User tapped widget add button")
Firebase.crashlytics.log("Midnight Reset Worker started")
Firebase.crashlytics.log("Attempting to schedule next hydration reminder")
```

---

## 6. Strict Privacy Rules
*   ❌ **NEVER** log the user's exact weight or gender to Firebase Analytics. (Calculate the goal locally, and only log the bucketted goal amount).
*   ❌ **NEVER** transmit specific timestamps of drinks off-device.
