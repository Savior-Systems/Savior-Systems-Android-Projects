# 06. AdMob Monetization Map — Water Log & Remind
```
Health and fitness apps require a delicate approach to monetization. Users open a water tracker to perform a 2-second action (log a drink). Interrupting this micro-interaction with a full-screen interstitial will lead to immediate uninstalls. 
```
Our strategy focuses on passive visibility and carefully gated interstitials.
```
---
```
## 1. Ad Format Strategy
```
### 1.1 Adaptive Banner Ads (Primary Passive Revenue)
*   **Placement A (Dashboard)**: Pinned to the bottom of the main dashboard screen, below the quick-add buttons and above the bottom navigation bar.
*   **Placement B (History)**: Pinned to the bottom of the History/Analytics tab.
*   **Behavior**: Auto-refreshes every 30 seconds.
*   **UX Safety**: Never obscures the main liquid progress circle or the quick-add buttons.
```
### 1.2 Interstitial Ads (Secondary Gated Revenue)
*   **Frequency Cap**: 1 impression per 3 hours per user.
*   **Trigger Points (Strictly Enforced)**:
    1.  **Exiting the History Screen**: Triggered when the user taps 'Back' to return to the Dashboard after viewing their analytics.
    2.  **Exiting Settings**: Triggered when leaving the Settings screen.
    3.  **The "5th Log" Milestone**: If the user has logged 5 drinks in a single day, an interstitial can fire *after* the 5th log is saved and the progress animation finishes.
*   **Absolute Restriction**: NEVER trigger an interstitial upon app open. NEVER trigger an interstitial instantly when a user taps a "Quick Add" button (it disrupts the dopamine hit of the liquid filling up).
```
### 1.3 Native Advanced Ads (Phase 2)
*   **Placement**: Embedded inside the "Daily Timeline" `LazyColumn` on the History screen (e.g., inserted as every 10th item).
*   **Styling**: Customized to match the app's Ocean Blue Material 3 card theme.
```

```
## 2. AdMob Settings & Content Filtering
```
*   **Category Blocks**: Block sensitive categories including Dating, Gambling, Alcohol (inappropriate for a health app), and Fast Food (conflicts with health/diet goals).
*   **eCPM Floor**: Maintain standard Google optimized floors for Tier-1 markets.
```

## 3. Implementation Logic (AdManager Singleton)
```
```kotlin
object WaterAdManager {
    private const val INTERSTITIAL_COOLDOWN_MS = 10800000L // 3 hours
    private const val LOGS_BEFORE_AD = 5
```
    fun shouldShowInterstitialAfterLog(dailyLogCount: Int): Boolean {
        val lastShown = UserPreferences.getLastInterstitialTime()
        val currentTime = System.currentTimeMillis()
```
        // Only consider showing if cooldown has passed AND they hit the log threshold
        if (currentTime - lastShown > INTERSTITIAL_COOLDOWN_MS && dailyLogCount % LOGS_BEFORE_AD == 0) {
            return true
        }
        return false
    }
}
```

## 4. In-App Purchase (IAP) Upsell
*   **Remove Ads ($2.99 - $4.99)**: A one-time purchase to permanently remove banner and interstitial ads. Prominently featured in the Settings menu and occasionally as a "Snack-bar" notification after a user achieves a 7-day streak.
