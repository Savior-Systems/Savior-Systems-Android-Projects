## 3. The Goal Calculator Engine (`GoalCalculatorEngine`)

Handles the logic for determining how much a user should drink.

```kotlin
class GoalCalculatorEngine @Inject constructor() {
    
    // Returns recommended daily goal in ml
    fun calculateGoal(
        weightKg: Double, 
        gender: Gender, 
        activityLevel: ActivityLevel,
        hotClimate: Boolean = false
    ): Int {
        // Base: ~33ml per kg of body weight
        var baseGoal = weightKg * 33.0
        
        // Activity modifiers
        val activityModifier = when(activityLevel) {
            ActivityLevel.SEDENTARY -> 0.0
            ActivityLevel.MODERATE -> 350.0 // Add 350ml
            ActivityLevel.ACTIVE -> 700.0   // Add 700ml
        }
        
        baseGoal += activityModifier
        
        // Gender modifier (Male generally requires slightly more)
        if (gender == Gender.MALE) {
            baseGoal *= 1.05
        }
        
        // Climate modifier
        if (hotClimate) {
            baseGoal *= 1.15
        }
        
        return baseGoal.roundToInt()
    }
}
```

## 4. Jetpack Glance Widget Architecture

Jetpack Glance allows us to build widgets using Compose-style syntax.

1.  **UI Construction**: `WaterGlanceWidget` creates the UI. It reads the current intake progress directly from a shared preference or repository flow.
2.  **Action Handling**: `actionRunCallback<AddWaterActionCallback>()` is attached to the widget's "Quick Add" buttons.
3.  **Background Execution**: `AddWaterActionCallback` executes in the background. It injects the `IntakeRepository`, adds the specified amount (e.g., 250ml) to the Room database, and then forces a widget UI update (`WaterGlanceWidget().updateAll(context)`).

## 5. Reminder System (WorkManager vs AlarmManager)

To ensure reliable reminders without draining battery:
*   **WorkManager (PeriodicWorkRequest)**: Used for the `MidnightResetWorker`. It runs once a day to evaluate streaks and reset daily counters.
*   **AlarmManager (setExactAndAllowWhileIdle)**: Used for the actual water reminders. Because health notifications are time-sensitive (e.g., users expect an alert exactly at 10:00 AM, not whenever the OS decides to batch jobs), `AlarmManager` is required to wake the device from Doze mode during the user's active waking hours.
