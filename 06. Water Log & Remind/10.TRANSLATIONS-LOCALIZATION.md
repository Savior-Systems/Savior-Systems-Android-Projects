## 3. Localization Best Practices

### 3.1 Unit Formatting
*   The UI must seamlessly toggle between Milliliters (ml) and Fluid Ounces (oz).
*   Internally, **the database always stores values in milliliters (ml)** to prevent rounding errors.
*   Use a centralized `UnitConverter` utility object for all UI rendering.
    ```kotlin
    fun formatVolume(mlAmount: Int, useOz: Boolean): String {
        return if (useOz) {
            val oz = mlAmount / 29.5735
            String.format(Locale.getDefault(), "%.0f oz", oz)
        } else {
            "$mlAmount ml"
        }
    }
    ```

### 3.2 Plurals Handling
