## 3. Key User Journeys
`
### 3.1 First Launch & Smart Goal Onboarding
1.  User opens app. Splash screen shows.
2.  App checks `DataStore` for a saved `daily_goal_ml`. If missing, routes to Onboarding.
3.  **Onboarding Step 1:** User inputs weight (kg/lbs) and gender.
4.  **Onboarding Step 2:** User selects activity level (Sedentary, Moderate, Active) and typical Wake/Sleep times.
5.  **Onboarding Step 3:** App calculates the recommended daily goal using standard hydration formulas (e.g., Weight in kg * 33ml + Activity modifier). User can manually override this number.
6.  User taps "Start Tracking". App saves profile to DataStore, initializes AlarmManager intervals based on Wake/Sleep times, and navigates to Dashboard.
`
### 3.2 Frictionless Jetpack Glance Widget Logging
1.  User adds the app widget to their Android Home Screen.
2.  Widget displays current progress bar (e.g., 500/2000ml) and a `+250ml` button.
3.  User taps `+250ml` directly on the home screen.
4.  Widget triggers an `ActionCallback`. The background process injects the log into the Room database.
5.  The Widget updates its UI state immediately. The main app is never opened, ensuring a zero-friction experience.
`
### 3.3 The Smart Reminder Loop
1.  User has wake time set to 07:00 and sleep time to 23:00, with a 2-hour reminder interval.
2.  `WorkManager` or `AlarmManager` schedules alarms at 09:00, 11:00, 13:00, etc.
3.  At 11:00, the system checks the user's progress for the day. If they have already met >50% of their goal, the notification might say: "Great job! Have another glass to keep the momentum going." If they are behind, it says: "You're falling behind! Time for a big glass of water."
4.  At 23:00, the app enters "Quiet Hours". No alarms will fire until 07:00 the next day.
`
### 3.4 Midnight Reset
1.  A `WorkManager` daily periodic task is scheduled to run exactly at 00:00 (midnight local time).
2.  The task resets the "current day's intake" UI state to 0.
3.  It evaluates the previous day's total. If the goal was met, it increments the user's `current_streak` in DataStore. If missed, it resets the streak to 0.

## 4. State Management Rules
`
### 4.1 Reactive Hydration State
*   The `DashboardViewModel` observes a Kotlin `Flow` from the `IntakeDao` representing `SUM(amount) WHERE date = today`.
*   Whenever the user adds a drink (via app or widget), the Room database updates, the Flow emits a new value, and the Compose UI automatically animates the water level rising. No manual UI refreshing is necessary.
