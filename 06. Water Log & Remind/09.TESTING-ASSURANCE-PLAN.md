# 09. Testing & Quality Assurance Plan — Water Log & Remind

## 1. Unit Testing Strategy (JUnit + MockK)

| Component | Test Objective |
| :--- | :--- |
| `GoalCalculatorEngine` | Input: 70kg, Male, Moderate Activity. Assert output equals expected base formula goal (e.g., ~2660ml). |
| `GoalCalculatorEngine` | Input: hot climate toggle ON. Assert output is exactly 15% higher than base calculation. |
| `HydrationFactorEngine`| Input: 250ml Coffee. Assert effective hydration returned is 200ml (assuming 0.8x factor). |

## 2. Integration Testing

| Component | Test Objective |
| :--- | :--- |
| `IntakeDao` | Insert 5 logs for today. Query `getTodaysTotalFlow()`. Assert the sum matches the exact total of the 5 logs. |
| `MidnightResetWorker` | Mock system time to 00:01. Trigger Worker manually via `WorkManagerTestInitHelper`. Assert a new record is created in `daily_summaries` and the streak is updated in DataStore. |

## 3. UI/Instrumentation Testing (Espresso / Compose Test)

| Screen | Test Objective |
| :--- | :--- |
| `OnboardingScreen` | Enter weight 80, select Male. Tap Next. Assert the final Goal screen displays the correct calculated number. |
| `DashboardScreen` | Tap the "+250ml" preset button. Assert the progress circle text updates from "0" to "250". |
| `HistoryScreen` | Swipe left on a history log item. Assert the delete button appears. Tap delete and assert the item is removed from the list. |

## 4. Manual QA Matrix & Edge Cases

| Test Case | Steps | Expected Result |
| :--- | :--- | :--- |
| **Widget Integration** | Add Jetpack Glance widget to home screen. Tap +200ml button on widget. | Widget text updates to +200. Open app -> App dashboard also reflects the +200ml addition. |
| **Timezone / DST Travel** | Log water at 11:00 PM. Change device timezone so it is now 01:00 AM the next day. | The dashboard resets to 0ml. The previous day's log is preserved in the history calendar. |
| **Quiet Hours Override** | Set wake time to 08:00 and sleep time to 22:00. Set device time to 23:30. Wait for a reminder interval. | No notification is fired during the sleep window. |
| **Doze Mode Alarms** | Set device to Airplane mode, unplug from USB, turn off screen. Wait for scheduled interval. | Reminder notification sounds exactly at the interval mark, successfully bypassing Doze restrictions. |

## 5. Performance Benchmarks

*   **Widget Latency:** Tapping a widget button must update the local Room database and refresh the widget UI in < 400ms to feel responsive.
*   **Cold Start Time:** < 600ms to a fully rendered Dashboard.
*   **Animation Framerate:** The `LiquidProgressCircle` Canvas animation must maintain 60fps on mid-range devices. Use `remember` and state hoisting to prevent unnecessary recompositions of the rest of the screen.
