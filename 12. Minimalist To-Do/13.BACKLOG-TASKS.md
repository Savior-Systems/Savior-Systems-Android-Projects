# 13. Backlog Tasks

```
This document contains the development roadmap and task breakdown for **Minimalist To-Do**, structured into 5 phased sprints.
```

```
## Sprint 1: Core Architecture & Database Setup (Days 1-2)
*   [ ] Initialize Android project and set up Gradle dependencies (Room, DataStore, Compose, Google Play Services Ads).
*   [ ] Build Room SQLite database schemas: `CategoryEntity` and `TaskEntity`.
*   [ ] Implement `TaskDao` with Flow-based active tasks and categories queries.
*   [ ] Configure local `DataStore` repository for caching selected filters and preferences.
*   [ ] Write unit tests verifying task database insertions, swipe updates, and cascade deletions.
```
## Sprint 2: Core UX & Quick Capture (Days 3-4)
*   [ ] Set up Compose theme system using primary **Charcoal Gray** (`#37474F`) and **Soft Teal** (`#4DB6AC`).
*   [ ] Build main checklist UI list layout with Compose components.
*   [ ] Build the anchored bottom quick-add task capture text field, focusing instantly on tap.
*   [ ] Implement top Category horizontal tab chips with active selection animations.
```
## Sprint 3: Gestures & Micro-interactions (Days 5-6)
*   [ ] Integrate Material 3 `SwipeToDismissBox` into task list items.
*   [ ] Connect Swipe-Right gesture to update task status in Room (`is_completed = 1`).
*   [ ] Connect Swipe-Left gesture to trigger `deleteTask()` query in Room.
*   [ ] Create scale-up entry animations for icons and strikethrough transition effects for completed tasks.
```
## Sprint 4: Monetization & Settings (Days 7-8)
*   [ ] Implement anchored Adaptive Banner Ad at the bottom of the main activity screen.
*   [ ] Build Settings dialog/screen overlay with options (Theme selector, Export/Import, Clear completed tasks).
*   [ ] Implement `AdMobHelper` featuring a strict **180-second global cooldown** guard for interstitial ads.
*   [ ] Trigger Interstitial Ad upon clearing completed tasks or exiting settings back to the main list.
```
## Sprint 5: QA, Localization & Publishing (Days 9-10)
*   [ ] Localize UI copy: Add string tables for Spanish (`es`) and Bengali (`bn`).
*   [ ] Run Play Store Policy compatibility checklist verification (verify precise alarm runtime requests).
*   [ ] Perform build profile optimization: Enable ProGuard / R8 code shrinking configurations.
*   [ ] Create production-ready release AAB (Android App Bundle) signed package for deployment.
```
