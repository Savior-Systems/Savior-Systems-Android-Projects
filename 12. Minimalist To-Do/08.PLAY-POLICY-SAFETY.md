# 08. Play Store Policy & Data Safety
```
This document details Google Play Console policy compliance guidelines, permission declarations, and the data safety profile for **Minimalist To-Do**.
```

```
## 1. Requested Permissions
To maximize user trust and prevent automated app flags during review, the app requests **zero invasive permissions**.
```
| Permission | API Level | Purpose | Necessity |
| :--- | :--- | :--- | :--- |
| `android.permission.POST_NOTIFICATIONS` | API 33+ | Show scheduled task reminders. | **Required** for push alerts. Opt-in at runtime. |
| `android.permission.SCHEDULE_EXACT_ALARM` | API 31+ | Precision scheduling of task reminders. | **Required** for alarms to fire exactly on time. |
```
No location, contact, phone, storage (using internal DB files), or hardware access permissions are declared.
```
---
```
## 2. Google Play Data Safety Declarations
```
### A. Data Collection
*   **Data Collected**: None.
*   The application does not transmit any user data (e.g., tasks, categories, settings) to external servers. Everything is stored locally in the Room SQLite database.
*   *Note*: If Firebase Analytics is enabled, it only tracks non-PII events (e.g., total task count metrics, screen transitions). No task titles or contents are ever collected.
```
### B. Data Sharing
*   **Data Shared**: None.
*   No personal data, device identifiers, or task contents are shared with third-party networks or service providers.
```
### C. Data Security Measures
*   **Encryption**: All data is encrypted in transit if Firebase analytics reports any metrics, but no core user task database data ever leaves the device.
*   **Data Deletion**: Users can delete their data at any time by:
    1.  Tapping "Delete All Tasks" in Settings.
    2.  Clearing the application cache/data in Android system settings.
```
