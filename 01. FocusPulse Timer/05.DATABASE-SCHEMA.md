# 05. Database Schema & DataStore
```
FocusPulse Timer uses Room Database for structured offline logging of timer sessions, and Jetpack DataStore (Preferences) for lightweight app configurations.
```
## 1. Room Database: Timer Sessions
```
### Entity: `SessionLog`
Logs every completed Pomodoro and break session.
```
```kotlin
@Entity(tableName = "session_logs")
data class SessionLog(
    @PrimaryKey(autoGenerate = true)
    val id: Long = 0,
```
    @ColumnInfo(name = "session_type")
    val sessionType: String, // "FOCUS", "SHORT_BREAK", "LONG_BREAK"
```
    @ColumnInfo(name = "duration_minutes")
    val durationMinutes: Int,
```
    @ColumnInfo(name = "timestamp_start")
    val timestampStart: Long,
```
    @ColumnInfo(name = "timestamp_end")
    val timestampEnd: Long,
```
    @ColumnInfo(name = "completed_successfully")
    val completedSuccessfully: Boolean
)
```
```
### DAO: `SessionDao`
```kotlin
@Dao
interface SessionDao {
    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insertSession(session: SessionLog)
```
    @Query("SELECT * FROM session_logs ORDER BY timestamp_end DESC")
    fun getAllSessionsFlow(): Flow<List<SessionLog>>
```
    @Query("SELECT SUM(duration_minutes) FROM session_logs WHERE session_type = 'FOCUS' AND timestamp_start >= :todayStart")
    fun getTodayFocusMinutesFlow(todayStart: Long): Flow<Int?>
```
    @Query("DELETE FROM session_logs")
    suspend fun clearAllHistory()
}
```
```
## 2. Jetpack DataStore (Preferences)
```
Lightweight key-value storage for app settings.
```
| Key | Type | Default Value | Description |
|---|---|---|---|
| `pref_focus_duration` | Int | 25 | Focus timer duration in minutes. |
| `pref_short_break_duration` | Int | 5 | Short break duration in minutes. |
| `pref_long_break_duration` | Int | 15 | Long break duration in minutes. |
| `pref_auto_start_breaks` | Boolean | false | Automatically start break when focus ends. |
| `pref_sound_theme` | String | "chime" | Selected alarm sound ID. |
| `pref_dark_mode` | Boolean | true | App theme mode preference. |
```
## 3. Migration Strategy
*   **Version 1**: Initial release schema as defined above. No migrations required yet. 
*   **Export Schema**: Enabled in Room database configuration for future automated migrations.
```
