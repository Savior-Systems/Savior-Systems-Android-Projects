## 2. Ad Placement Rules & Triggers

### 2.1 App Open Ad — `AppOpenAdManager`
*   **Trigger**: App cold start AND background return (app was backgrounded > 60 seconds).
*   **Cooldown**: Hard cap of **60,000ms (60 seconds)**. Never show twice within 60 seconds.
*   **Timing**: Loads silently during splash screen. Displays if loaded when main activity resumes. If not loaded yet → proceed to main screen, load in background.
*   **Expected eCPM**: $7.00–$10.00 (US/UK), $0.50–$1.50 (BD/IN).
*   **Anti-IVT note**: Do NOT show on every onResume(). Gate strictly behind the 60s cooldown and only on user-initiated app opens (not notification taps from background).

### 2.2 Adaptive Banner — `AdBannerComposable`
*   **Trigger**: Always visible on `TimerScreen` and `HistoryScreen`. Loaded on screen creation.
*   **Lifecycle**: Attach banner lifecycle to `DisposableEffect` in Compose — destroy on screen disposal.
*   **Position**: Anchored at absolute bottom, never overlapping any interactive controls.
*   **Expected eCPM**: $0.50–$1.50 (Tier-1), $0.10–$0.40 (South Asia).
*   **Collapsible variant**: Use `AdSize.getCurrentOrientationAnchoredAdaptiveBannerAdSize()` for adaptive sizing.

### 2.3 Interstitial — `AdManager`
*   **Trigger**: Work session completes → Focus Card bottom sheet → user taps "Share" OR "Skip" → interstitial shown (if 180s cooldown passed).
*   **Cooldown**: Hard cap of **180,000ms (3 minutes)**. Enforced programmatically via `lastAdShowTime` timestamp.
*   **Flow**: Card share/skip → check cooldown → if passed: show interstitial → on dismiss: navigate to Break screen. If cooldown not passed: navigate to Break screen directly.
*   **Prefetch**: Load new interstitial immediately after each one is shown.
*   **Expected eCPM**: $5.00–$8.00 (Tier-1), $0.30–$1.00 (South Asia).

### 2.4 Rewarded Ad — Sound Unlock
*   **Trigger**: User taps a locked soundscape (White Noise, Forest, Cafe, Fireplace) on Settings screen.
*   **Dialog before ad**: "Watch a short video to unlock [Sound Name] for 24 hours" — YES / CANCEL.
*   **On reward granted**: Set `SOUND_UNLOCK_EXPIRY = currentTime + 86_400_000L` in DataStore. Unlock all 4 premium sounds for 24 hours.
*   **Expected eCPM**: $15.00–$25.00 (Tier-1), $1.00–$3.00 (South Asia).

### 2.5 Rewarded Ad — Streak Shield
*   **Trigger**: User taps shield badge on Timer screen → Shield Bottom Sheet → "Earn a Shield" button.
*   **Cooldown**: One rewarded shield grant per 24 hours per device.
*   **Check before showing ad**: `currentTime - lastRewardedDate < 86_400_000L` → show "Come back tomorrow for another shield".
*   **On reward granted**: `ShieldDao.addRewardedShield(currentTime)`. Max shields = 2.
*   **Expected eCPM**: $15.00–$25.00 (Tier-1).

### 2.6 Rewarded Ad — Card Theme Unlock
*   **Trigger**: User taps "Bold" or "Dark" card template preview in Settings or in the Focus Card bottom sheet.
*   **Dialog**: "Watch a short video to permanently unlock this card design" — YES / CANCEL.
*   **On reward granted**: Set `CARD_BOLD_UNLOCKED = true` OR `CARD_DARK_UNLOCKED = true` in DataStore. Permanent — never expires.
*   **Expected eCPM**: $15.00–$25.00 (Tier-1).

### 2.7 Rewarded Ad — Weekly Focus Boost
*   **Trigger**: Shown on Weekly Challenge completion as an optional bonus: "Watch a video for a bonus 7-day sound unlock on top of your reward".
*   **Context**: User just completed the challenge and is in a positive emotional state — highest conversion moment.
*   **On reward granted**: Set `CARD_WEEKLY_UNLOCK_EXPIRY = currentTime + 604_800_000L` (7 days) in DataStore.
*   **Expected eCPM**: $15.00–$25.00 (Tier-1).

## 4. AdMob Invalid Traffic (IVT) Protection Checklist

- [x] Interstitial: 180s hard cooldown enforced via `System.currentTimeMillis()` delta.
- [x] App Open: 60s hard cooldown. Never shown on notification-triggered resumes.
- [x] Rewarded Sound: Gated behind user intent dialog. Never auto-shown.
- [x] Rewarded Shield: 24h per-device cooldown enforced via DataStore timestamp.
- [x] Never show any ad format during active timer countdown (this would be disruptive and flagged).
- [x] All ad calls go through a single `AdManager` or `AppOpenAdManager` singleton — no duplicate ad instances.
- [x] Banner lifecycle tied to Compose `DisposableEffect` — destroyed on screen exit.
- [x] UMP Consent (GDPR) check enforced on first launch before any ad loads.

    buildFeatures { buildConfig = true }
    defaultConfig {

    implementation("com.google.android.gms:play-services-ads:23.x.x")

    android:name="com.google.android.gms.ads.APPLICATION_ID"
    android:value="${admobAppId}"/>
