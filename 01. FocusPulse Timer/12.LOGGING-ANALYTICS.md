## 2. Firebase Analytics Event Schema

All events logged via a centralized `Analytics.kt` wrapper to maintain consistent naming and prevent duplicate logging.

```kotlin
object Analytics {
    fun log(event: String, params: Bundle = Bundle()) =
        FirebaseAnalytics.getInstance(App.context).logEvent(event, params)
}
```

---

## 3. Timer Events

### `timer_started`
Fired when user initiates a new Focus Session.
```kotlin
Analytics.log("timer_started", bundleOf(
    "session_type" to "WORK",          // "WORK" | "SHORT_BREAK" | "LONG_BREAK"
    "duration_minutes" to 25,
    "cycle_index" to 0                 // 0–3
))
```

### `timer_paused`
Fired when user taps Pause during an active session.
```kotlin
Analytics.log("timer_paused", bundleOf(
    "time_remaining_seconds" to 720,
    "session_type" to "WORK"
))
```

### `timer_completed`
Fired when session countdown reaches 00:00.
```kotlin
Analytics.log("timer_completed", bundleOf(
    "session_type" to "WORK",          // "WORK" | "SHORT_BREAK" | "LONG_BREAK"
    "duration_minutes" to 25,
    "sessions_today" to 3              // total WORK sessions completed today
))
```

### `timer_reset`
Fired when user manually resets the timer.
```kotlin
Analytics.log("timer_reset", bundleOf(
    "time_remaining_seconds" to 480,
    "session_type" to "WORK"
))
```

## 4. 🆕 Focus Card Events

### `focus_card_generated`
Fired when the app generates a Focus Card bitmap after session completion.
```kotlin
Analytics.log("focus_card_generated", bundleOf(
    "card_template" to "MINIMAL",      // "MINIMAL" | "BOLD" | "DARK"
    "total_focus_minutes_today" to 75,
    "streak_days" to 14
))
```

### `focus_card_shared`
Fired when user taps "Share My Focus" and the system share intent is dispatched.
```kotlin
Analytics.log("focus_card_shared", bundleOf(
    "card_template" to "BOLD",
    "platform" to "whatsapp"           // Inferred from resolved app if possible; else "other"
))
```

### `focus_card_skipped`
Fired when user taps "Skip" on the Focus Card bottom sheet.
```kotlin
Analytics.log("focus_card_skipped", bundleOf(
    "card_template" to "MINIMAL"
))
```

Fired when user starts a timer via widget action button (not from in-app).

```kotlin
    "widget_size" to "2x1"
))
```

## 6. 🆕 Streak & Shield Events

### `streak_shield_activated`
Fired when a shield consumes to protect streak (auto-consumed or user-initiated restore).
```kotlin
Analytics.log("streak_shield_activated", bundleOf(
    "type" to "auto",                  // "auto" | "restore_rewarded"
    "streak_preserved" to 14
))
```

### `streak_broken`
Fired when streak resets to 0 (no shield available, day was missed).
```kotlin
Analytics.log("streak_broken", bundleOf(
    "streak_at_break" to 7             // What the streak was before breaking
))
```

### `milestone_unlocked`
Fired when a milestone badge is earned.
```kotlin
Analytics.log("milestone_unlocked", bundleOf(
    "badge_id" to "STREAK_7"          // "EARLY_BIRD" | "DEEP_WORKER" | "STREAK_3" | "STREAK_7" | "STREAK_30" | "WEEKLY_WARRIOR"
))
```

## 7. 🆕 Weekly Challenge Events

### `weekly_challenge_started`
Fired when a new weekly challenge is created (Monday reset detected).
```kotlin
Analytics.log("weekly_challenge_started", bundleOf(
    "target_sessions" to 20,
    "difficulty" to "STANDARD"         // "BEGINNER" | "STANDARD" | "ADVANCED"
))
```

### `weekly_challenge_completed`
Fired when user completes all required sessions in the current challenge.

```kotlin
Analytics.log("weekly_challenge_completed", bundleOf(
    "target_sessions" to 20,
    "days_taken" to 5,                 // How many calendar days it took
    "difficulty" to "STANDARD"
))
```

## 9. Ad Events

### `ad_shown`
Fired whenever any ad format is displayed.
```kotlin
Analytics.log("ad_shown", bundleOf(
    "ad_type" to "INTERSTITIAL",       // "INTERSTITIAL" | "APP_OPEN" | "BANNER_IMPRESSION" | "REWARDED"
    "trigger_context" to "session_complete" // "session_complete" | "cold_start" | "background_return" | "sound_unlock" | "shield_earn" | "card_theme" | "challenge_boost"
))
```

### `rewarded_ad_completed`
Fired when user completes watching a rewarded ad (reward granted).
```kotlin
Analytics.log("rewarded_ad_completed", bundleOf(
    "reward_type" to "SOUND_UNLOCK",   // "SOUND_UNLOCK" | "STREAK_SHIELD" | "CARD_BOLD" | "CARD_DARK" | "WEEKLY_BOOST"
    "sound_type" to "WHITE_NOISE"      // Only when reward_type = "SOUND_UNLOCK"
))
```

## 10. Retention Cohort Tracking

Firebase automatically tracks D1/D7/D30 retention if `first_open` event is present. Supplement with:

### `app_opened`
Log on every app open (beyond Firebase's automatic `app_open`) to enable custom cohort funnels.
```kotlin
Analytics.log("app_opened", bundleOf(
    "is_cold_start" to true,
    "streak_at_open" to 14,
    "sessions_today_at_open" to 0
))
```

## 11. Firebase Crashlytics Keys

Set custom keys on session start to improve crash context:
```kotlin
FirebaseCrashlytics.getInstance().apply {
    setCustomKey("timer_state", "ACTIVE")          // Current timer state
    setCustomKey("streak_days", 14)                // Current streak
    setCustomKey("sessions_today", 2)              // Sessions completed today
    setCustomKey("db_schema_version", 2)           // Room DB version
    setCustomKey("locale", Locale.getDefault().language)
}
```
