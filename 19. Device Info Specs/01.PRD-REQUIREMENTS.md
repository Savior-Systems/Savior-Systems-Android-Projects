# 01. Product Requirement Document (PRD)

## 1. Vision & Strategy
*   **App Name**: Device Info Specs
*   **Category**: Tools
*   **Core Objective**: Build a fast, offline device specs diagnostic dashboard. It targets search intent for: `system monitor`, `device hardware info`, and `cpu specs`.
*   **Value Proposition**: Instant, offline hardware specifications (CPU, RAM, GPU, sensors, and battery metrics) with zero internet access permissions.

---

## 2. Target Persona
### Persona: Ethan (Tech Enthusiast & Used Phone Buyer)
*   **Needs**: Open an app on a potential used phone purchase to instantly verify true hardware capabilities (RAM, screen resolution, camera sensors, and battery health) offline.
*   **Pain Points**: Heavy diagnostic tools filled with ads, requiring internet access to run, or tracking location.
*   **How the App Solves it**: The application requests zero internet permissions, mapping all local hardware configurations directly from the Android System APIs instantly.

---

## 3. Core Feature List
1.  **System Dashboard**:
    *   Real-time CPU cores speed dials.
    *   RAM memory usage gauge (Total vs Free).
    *   Battery health, temperature, and charging speed readouts.
2.  **Hardware Specs Detail Tabs**:
    *   Display info (Density, Resolution, Refresh Rate).
    *   Storage specifications.
    *   Camera sensor megapixel readouts.
3.  **Sensor Audit Checker**:
    *   Live testing grids for sensors (Accelerometer, Gyroscope, Magnetometer).

---

## 4. Monetization & Ad Zones
*   **Zero-Ad Zones**:
    *   Real-time system sensor grids and CPU core monitor charts are kept 100% ad-free.
*   **Monetization**:
    *   Banner ad at the bottom of the main dashboard.
    *   Interstitial ad on initiating a full hardware diagnostics sensor test, guarded by a **180s cooldown**.
