# 08. Play Store Policy & Data Safety
```
This document outlines security compliance properties for **WiFi QR Sharer**.
```
---
```
## 1. Requested Permissions
To maximize user trust and prevent automated app flags during review, the app requests the following permission:
```
| Permission | API Level | Purpose | Necessity |
| :--- | :--- | :--- | :--- |
| `android.permission.CAMERA` | Any | Access camera view finder to scan WiFi QR codes. | **Required** for the scanning feature. Opt-in at runtime. |
```
No location, contact, phone, or storage permissions are declared.
```
---
```
## 2. Google Play Data Safety Declarations
```
### A. Data Collection
*   **Data Collected**: None.
*   The application does not collect or transmit any user data (e.g., SSID, passwords, scanned codes) to external servers.
```
### B. Data Sharing
*   **Data Shared**: None.
*   No personal data, device identifiers, or network parameters are shared with third-party networks or service providers.
```
### C. Data Security Measures
*   **Encryption**: No data leaves the device.
*   **Data Deletion**: Users can delete their saved history by clearing app storage in system settings.
