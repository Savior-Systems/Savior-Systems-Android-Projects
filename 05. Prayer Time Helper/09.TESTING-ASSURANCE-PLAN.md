# 09. Testing & Quality Assurance Plan — Prayer Time Helper
`
## 1. Unit Testing Strategy (JUnit + MockK)
`
| Component | Test Objective |
| :--- | :--- |
| `AdhanCalculationEngine` | Input a known location (e.g., Mecca) and known date. Assert that output times match expected official times within ±1 minute. |
| `AdhanCalculationEngine` | Assert that switching from Shafi'i to Hanafi significantly delays the Asr time output. |
| `HijriDateConverter` | Assert that 2024-03-11 correctly converts to 1 Ramadan 1445 (with standard offsets). |
| `QiblaCalculator` | Input Dhaka coordinates. Assert that output bearing is exactly 292.8°. |
`
## 2. Integration Testing
`
| Component | Test Objective |
| :--- | :--- |
| `PrayerCacheDao` | Insert 7 days of timings. Query `getUpcomingWeek()`. Assert list size is 7 and dates are ordered correctly. |
| `GeocodingService` | Mock Retrofit response for "Dhaka". Assert the repository correctly parses lat/long and saves to DataStore. |
`
## 3. UI/Instrumentation Testing (Espresso / Compose Test)
`
| Screen | Test Objective |
| :--- | :--- |
| `DashboardScreen` | Mock `DailyTimings`. Assert that Fajr, Dhuhr, Asr, Maghrib, Isha rows are visible and texts match the mock. |
| `SetupScreen` | Deny location permission. Assert that the "Manual City Search" field becomes visible and functional. |
| `TasbihScreen` | Tap the counter button 33 times. Assert the text reads "33". |
`
## 4. Manual QA Matrix & Edge Cases
`
| Test Case | Steps | Expected Result |
| :--- | :--- | :--- |
| **Compass Sensor Absence** | Run app on an Android Emulator (which lacks a real magnetometer). | Compass screen displays a graceful error message: "Compass sensor not detected on this device." |
| **Midnight Rollover** | Set device time to 23:59. Wait 1 minute. | Dashboard automatically refreshes to show tomorrow's timings without crashing or requiring a restart. |
| **Airplane Mode** | Turn on Airplane mode. Clear app cache. Launch app. Set location manually. | App fully calculates timings and displays dashboard without internet access. |
| **Doze Mode Alarm** | Set an Azan alarm 5 minutes into the future. Put device to sleep (screen off). | Screen wakes (or notification sounds) precisely at the minute mark, bypassing Doze. |
`
## 5. Performance Benchmarks
`
*   **Offline Calculation Speed:** `AdhanCalculationEngine` must generate a full month of timings in < 150ms on a mid-range device.
*   **Cold Start Time:** < 800ms to visible dashboard (assuming location is already cached).
*   **Compass Rendering:** UI thread must maintain 60fps while the compass needle is rapidly rotating. Use `remember` and lightweight state observation to avoid recomposing the entire screen.
`
