# 02. UI/UX Design System — Prayer Time Helper
`
This design system establishes a serene, modern, and spiritually resonant interface tailored for a religious utility app. The aesthetic is clean and distraction-free, minimizing cognitive load.
`
---
`
## 1. Color Palette (Material 3 Dynamic Mappings)
`
### Light Mode Color Scheme
*   **Primary (Brand):** `#00695C` (Deep Emerald Green — traditionally associated with Islam and tranquility)
*   **On Primary:** `#FFFFFF` (Crisp White)
*   **Primary Container:** `#B2DFDB` (Soft Mint)
*   **Secondary:** `#1565C0` (Midnight Blue — represents the night sky/Isha)
*   **Secondary Container:** `#BBDEFB` (Soft Sky)
*   **Tertiary (Accent):** `#F39C12` (Sunrise Gold — highlights active states and the sun)
*   **Background:** `#F8F9FA` (Clean Off-White)
*   **Surface:** `#FFFFFF` (Card surfaces)
*   **Outline:** `#CFD8DC` (Subtle borders)
`
### Dark Mode Color Scheme
*   **Primary (Brand):** `#4DB6AC` (Luminous Teal)
*   **On Primary:** `#003D33` (Very Dark Green)
*   **Primary Container:** `#004D40` (Rich Forest)
*   **Secondary:** `#64B5F6` (Starlight Blue)
*   **Tertiary (Accent):** `#F1C40F` (Bright Gold)
*   **Background:** `#121212` (Material Dark)
*   **Surface:** `#1E1E1E` (Elevated Surface)
*   **Outline:** `#37474F`
`

`
## 2. Typography Rules
*   **Primary Font Family:** `Outfit` (Modern, geometric, high legibility for English digits).
*   **Secondary Font Family:** `Amiri` or `Lateef` (Optional Google Fonts for rendering Arabic script beautifully, e.g., for "Allahu Akbar" or Hijri months).
*   **Alternative/Fallback:** `System Default Sans-Serif`.
`
### Size Mappings
| Style | Size | Weight | Usage |
| :--- | :--- | :--- | :--- |
| `Display Large` | 48sp | Bold | Next Prayer Countdown Timer |
| `Title Large` | 22sp | Semi-Bold | Screen titles, Prayer Names |
| `Title Medium` | 18sp | Medium | Times, Settings headers |
| `Body Large` | 16sp | Regular | General text, dates |
| `Label Medium` | 12sp | Medium | Badges (e.g., "Hanafi", "Karachi") |
`
---
`
## 3. Component Design Tokens
`
### 3.1 Daily Timings Card
*   **Corner Radius**: 16dp
*   **Elevation**: 0dp (flat design with a 1dp outline)
*   **Layout**: Horizontal row for each prayer (Name left, Time right, Alarm icon far right).
*   **Active State**: The current prayer row has a subtle Primary Container background fill and a bold font weight.
`
### 3.2 Next Prayer Header
*   **Background**: A gradient card sweeping from Primary (Emerald) to Secondary (Midnight Blue).
*   **Content**: "Next: Asr", large countdown timer "01:45:20", and the location string "Dhaka, BD".
*   **Text Color**: `#FFFFFF` for maximum contrast.
`
### 3.3 Qibla Compass
*   **Compass Dial**: Minimalist circular ring with degree markers.
*   **Needle**: Dual-tone needle pointing to Mecca (Gold head, Dark tail).
*   **Kaaba Icon**: A small, stylized cube icon marking the exact Qibla degree on the dial rim.
`
### 3.4 Tasbih Counter
*   **Touch Target**: A massive circular ripple button taking up 50% of the screen.
*   **Count Display**: Massive typography (`Display Large`) centered above the button.
`
---
`
## 4. Gesture Animations & Micro-Interactions
`
### 4.1 Compass Physics
*   **Rotation**: Interpolated, physics-based rotation. The needle should never "snap" instantly but rather swing with simulated friction/damping (using Compose `animateFloatAsState` with a spring spec).
*   **Haptic Alignment**: When the device bearing matches the Qibla bearing (±2 degrees), trigger a solid haptic pulse (`HapticFeedbackType.LongPress`) and change the dial color to Gold.
`
### 4.2 Alarm Toggles
*   **Icon Switch**: Tapping the alarm icon next to a prayer smoothly morphs between `VolumeOff`, `NotificationsActive` (Standard beep), and `VolumeUp` (Full Azan).
`
---
`
## 5. Adaptive Layout Rules
*   **Compact (< 600dp):** Standard vertical scrolling list, bottom navigation bar.
*   **Medium/Expanded (> 600dp):** Two-pane layout. Left pane: Next Prayer Header + Timings List. Right Pane: Qibla Compass + Tasbih.
`
