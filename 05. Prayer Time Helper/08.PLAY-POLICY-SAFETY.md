# 08. Play Store Policy & Safety — Prayer Time Helper
```
Prayer apps often face intense scrutiny from Google Play Reviewers due to excessive permission requests (especially Background Location) and alarm handling. This document details our compliance strategy.
```
---
```
## 1. Permissions Audit (`AndroidManifest.xml`)
```
### Required Permissions (Justified)
*   `INTERNET`: Required ONLY for serving AdMob ads and the one-time Geocoding manual city search.
*   `ACCESS_NETWORK_STATE`: Required by AdMob to check connection before requesting ads.
*   `VIBRATE`: Required for the Tasbih counter and Qibla compass alignment haptics.
*   `WAKE_LOCK`: Required to turn on the screen briefly when an exact Azan alarm fires.
*   `SCHEDULE_EXACT_ALARM`: Required (Android 12+) to guarantee Azan notifications trigger exactly on time.
*   `POST_NOTIFICATIONS`: Required (Android 13+) to show the Azan alert in the system tray.
*   `RECEIVE_BOOT_COMPLETED`: Required to reschedule AlarmManager intents after a device reboot.
```
### Conditional Permissions (Prompted at Runtime)
*   `ACCESS_COARSE_LOCATION` / `ACCESS_FINE_LOCATION`: Required *only* if the user chooses the "Auto-Locate" setup option.
```
### STRIPPED Permissions (DO NOT ADD)
*   🚫 `ACCESS_BACKGROUND_LOCATION`: **Strictly Prohibited.** We calculate times based on a static cached location. We do not track the user as they move.
*   🚫 `READ_EXTERNAL_STORAGE` / `WRITE_EXTERNAL_STORAGE`: Not needed.
```
---
```
## 2. Location Policy Compliance Strategy
```
Google will reject the app if it detects background location without an approved declaration form.
**Our Defense:**
1.  We only request Foreground Location (`ACCESS_COARSE_LOCATION`) on the Setup Screen.
2.  The location is fetched once, saved to DataStore, and the location client is immediately disconnected.
3.  We provide a manual "City Search" fallback. If reviewers test the app and deny the location permission, the app remains 100% functional via manual entry.
```
---
```
## 3. Alarm & Exact Notification Compliance
```
Starting Android 14, `SCHEDULE_EXACT_ALARM` is restricted to apps that act as calendars or alarm clocks.
**Our Defense:**
1.  Prayer Time Helper acts directly as an **Alarm Clock** for religious timings.
2.  If challenged during app review, we will state: *"The core functionality of this app is to serve as a precise alarm clock for 5 daily prayers. Users rely on exact, down-to-the-minute alarms (Azans) to begin their religious obligations. Inexact alarms would render the application useless."*
```
