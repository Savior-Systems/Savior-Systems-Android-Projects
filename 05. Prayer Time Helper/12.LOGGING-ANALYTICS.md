# 12. Logging & Analytics — Prayer Time Helper
```
This document defines the Firebase Analytics telemetry schema, custom event parameters, and Crashlytics configuration for app optimization. We prioritize privacy and collect **zero personally identifiable information (PII)** and **zero precise location data**.
```

## 1. Firebase Analytics Configuration
```
### 1.1 SDK Integration
*   **BOM Version**: Firebase BOM 33+ (latest stable).
*   **Dependencies**: `firebase-analytics-ktx`, `firebase-crashlytics-ktx`.
*   **Consent**: Analytics collection starts on first app launch.
```
### 1.2 Automatic Events (No Configuration Needed)
Firebase automatically logs these events:
*   `first_open` — First app launch after install.
*   `session_start` — Beginning of a new session.
*   `app_update` — First launch after app version update.
*   `os_update` — First launch after OS version change.

## 2. Custom Firebase Events
```
### 2.1 `location_setup_completed`
Triggered when the user successfully sets their location during onboarding.
*   **Parameters:**
    *   `setup_method`: String (`"gps_auto"`, `"manual_city"`)
    *   `country_code`: String (e.g., `"BD"`, `"PK"`, `"GB"` - Extracted from geocoder, NEVER exact lat/long)
*   **Purpose**: Understand onboarding drop-off and preferred setup method.
```
### 2.2 `calculation_method_changed`
Triggered when a user changes the Adhan calculation method in settings.
*   **Parameters:**
    *   `new_method`: String (`"KARACHI"`, `"MWL"`, `"ISNA"`, etc.)
*   **Purpose**: Identify the most popular regional methods to set better defaults in future updates.
```
### 2.3 `asr_madhab_changed`
Triggered when a user toggles between Hanafi and Shafi'i Asr calculation.
*   **Parameters:**
    *   `madhab`: String (`"HANAFI"`, `"SHAFI"`)
*   **Purpose**: Track regional fiqh preferences.
```
### 2.4 `alarm_configured`
Triggered when a user changes an alarm setting for a specific prayer.
*   **Parameters:**
    *   `prayer_name`: String (`"fajr"`, `"dhuhr"`, `"asr"`, `"maghrib"`, `"isha"`)
    *   `alarm_type`: String (`"silent"`, `"beep"`, `"full_azan"`)
*   **Purpose**: Measure which prayers users most frequently set active alarms for.
```
### 2.5 `compass_calibration_prompted`
Triggered when the device sensor accuracy drops to `UNRELIABLE` and the calibration UI is shown.
*   **Parameters:**
    *   `action_taken`: String (`"calibrated"`, `"ignored"`)
*   **Purpose**: Monitor hardware sensor reliability across the Android ecosystem.
```
### 2.6 `tasbih_session_completed`
Triggered when a user resets the Tasbih counter after counting.
*   **Parameters:**
    *   `final_count_range`: String (bucketed: `"0-33"`, `"34-99"`, `"100-500"`, `"500+"`)
*   **Purpose**: Measure engagement with the bonus Tasbih feature.
```

## 5. Crashlytics Configuration
```
### 5.1 Crash Reporting
*   **Automatic**: All uncaught exceptions are captured and uploaded on next session.
*   **Non-Fatal Logging**: Use `Firebase.crashlytics.recordException(e)` for caught exceptions in `AdhanCalculationEngine`, `GeocodingService`, or `SensorManager` failures.
```
### 5.2 Custom Keys for Crash Context
```kotlin
Firebase.crashlytics.setCustomKey("has_magnetometer", sensorManager.getDefaultSensor(Sensor.TYPE_MAGNETIC_FIELD) != null)
Firebase.crashlytics.setCustomKey("calc_method", currentCalcMethod)
Firebase.crashlytics.setCustomKey("can_schedule_exact_alarms", alarmManager.canScheduleExactAlarms())
```
```
### 5.3 Breadcrumbs
Log non-fatal breadcrumbs for debugging crash context:
```kotlin
Firebase.crashlytics.log("User changed location via manual search")
Firebase.crashlytics.log("Attempting to schedule exact alarm for Fajr")
Firebase.crashlytics.log("Sensor accuracy changed to UNRELIABLE")
```

## 6. Strict Privacy & Policy Compliance
```
### 6.1 No Location Tracking Rules
*   ❌ **NEVER** log latitude or longitude coordinates to Firebase Analytics.
*   ❌ **NEVER** log precise location to Crashlytics logs.
*   ✅ Only log the high-level `country_code` (e.g., `"BD"`) returned by the geocoder to understand demographic distribution.
```
### 6.2 Data Retention
*   Firebase Analytics: 14-month rolling retention.
*   Crashlytics: 90-day crash log retention.
