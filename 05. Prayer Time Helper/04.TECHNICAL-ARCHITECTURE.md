## 4. Alarm System Architecture (Critical for Reliability)
```
Android's battery optimizations (Doze mode) routinely kill background apps. To ensure Azan alarms fire exactly on time:
```
1.  **Scheduling**: Use `AlarmManager.setExactAndAllowWhileIdle()` (API 23+) or `setAlarmClock()` for precise triggering.
2.  **Receiver**: `AzanAlarmReceiver` catches the intent and immediately triggers a foreground notification with a full-screen intent (optional) and audio payload.
3.  **Boot Persistence**: Android clears AlarmManager on device reboot. `BootReceiver` listens for `ACTION_BOOT_COMPLETED` and immediately invokes the WorkManager to recalculate and re-register alarms for the upcoming days.

## 5. Compass Sensor Fusion
```
A raw magnetometer is too jittery for a smooth UI.
*   We use a `SensorManager.getRotationMatrix()` combining `TYPE_ACCELEROMETER` and `TYPE_MAGNETIC_FIELD`.
*   We apply a Low-Pass Filter (LPF) algorithm to the raw sensor data in `CompassViewModel` to smooth the needle rotation before emitting it to the Compose UI.
```
