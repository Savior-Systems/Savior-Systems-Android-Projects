# 01. Product Requirement Document (PRD) — Routine Widget
```
## 1. Vision & Strategy
*   **App Name**: Routine Widget
*   **Category**: Productivity / Utilities
*   **Core Objective**: Build a performant, 100% offline-first application that displays daily routines and task schedules directly on the Android home screen via interactive Jetpack Glance widgets.
*   **Target Market**: Busy individuals, students, and remote workers looking to build habits and stick to a daily routine without the friction of continually launching full apps.
```
---
```
## 2. Target Personas
```
### Persona A: Liam Davies (28, London, UK)
*   **Role**: Work-from-Home Software Engineer.
*   **Pain Point**: Often forgets micro-habits (drinking water, stretching, checking emails) during back-to-back coding sessions because they are buried inside separate planner apps.
*   **Usage**: Places the Routine Widget on his primary home screen. Marks items off instantly as he completes them throughout the day.
```
### Persona B: Adnan Chowdhury (19, Chittagong, Bangladesh)
*   **Role**: University Student.
*   **Pain Point**: Class schedules change daily, and he needs a fast, lightweight layout showing what class is next without opening a heavy calendar app.
*   **Usage**: Adds his weekly class routines to the app, relying on the home screen widget to show the current day's timeline automatically.
```
---
```
## 3. Core Feature List & User Stories
```
### Feature 1: Interactive Home Screen Glance Widget
*   **Requirement**: Build a 4x2 / 4x4 Android widget using `Jetpack Glance`. Display the current day's routine tasks with checkboxes. Allow marking items complete directly from the widget.
*   **User Story**: *As Liam, I want to check off a task on my home screen widget so that I don't have to launch the app to record completion.*
```
### Feature 2: Weekly Routine Template Scheduler
*   **Requirement**: In-app configuration dashboard. Create repeating daily routines (e.g. Morning, Evening) or assign specific tasks to days of the week (Monday–Sunday).
*   **User Story**: *As Adnan, I want to define different class routines for different days of the week so that the widget updates automatically each morning.*
```
### Feature 3: Local Daily Reset Engine
*   **Requirement**: A background worker that fires at midnight to archive completed tasks and reset checkboxes for the new day's routine.
*   **User Story**: *As Liam, I want my routine list to clear and show today's tasks when I wake up so that I don't see yesterday's checked items.*
```
---
```
## 4. Key Performance Indicators (KPIs)
*   **Widget Response Latency**: Tapping a widget checkbox must update the database and refresh the widget UI in `< 150ms`.
*   **Cold Start (Configurator)**: In-app editor loading time `< 400ms`.
*   **Battery Overhead**: Midnight reset worker must consume `< 1%` battery per day.
```
