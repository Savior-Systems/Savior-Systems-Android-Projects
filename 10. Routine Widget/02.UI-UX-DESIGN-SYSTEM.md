# 02. UI/UX Design System — Routine Widget
```
Warm, action-oriented design built with the Sunset Orange color scheme for high visibility.
```
---
```
## 1. Color System (Sunset Orange)
```
### Light Theme
*   **Primary (`#E65100`)**: Deep Sunset Orange for checkboxes, time headers, and add-task FAB buttons.
*   **Primary Container (`#FFE0B2`)**: Warm cream highlight background for current active items.
*   **Secondary (`#F57C00`)**: Bright orange accents.
*   **Tertiary (`#00897B`)**: Vibrant Teal for success and completion states.
*   **Background (`#FFFBF7`)**: Warm off-white page backing.
*   **Surface (`#FFFFFF`)**: Pure white cards for individual task entries.
*   **On-Background (`#2D1C10`)**: Deep brown-charcoal text.
```
### Dark Theme
*   **Primary (`#FFB74D`)**: Soft pastel orange for high contrast.
*   **Primary Container (`#E65100`)**: Rich orange container fill.
*   **Secondary (`#FFA726`)**: Light orange items.
*   **Tertiary (`#4DB6AC`)**: Soft teal.
*   **Background (`#15100C`)**: Warm dark brown-black.
*   **Surface (`#221A15`)**: Card background.
*   **On-Background (`#F5EBE6`)**: Crisp ivory text.
```
---
```
## 2. Typography Standard
*   **Font Family**: **Outfit** (via Google Fonts Integration) for geometric readability at small widget sizes.
*   **Title Medium (Task Item)**: 16sp, Semi-Bold, tracking 0.1px.
*   **Body Small (Time tag)**: 12sp, Medium, tracking 0.4px.
```
---
```
## 3. Micro-Animations
```
### Animation A: Checkbox Complete Bounce
*   **Trigger**: Tapping a task checkbox to mark it complete.
*   **Behavior**: The checkmark pops in using a custom bouncy spring (`scaleY` & `scaleX` from 0.0 to 1.25 to 1.0) over 300ms, while the task text color turns to teal and shifts slightly to indicate completion.
```
### Animation B: Widget Refresh Flash
*   **Trigger**: Database updates sync to widget layout.
*   **Behavior**: A subtle alpha flash (fade out to 0.9, fade back in to 1.0) over 150ms applied to the widget elements, confirming the UI sync.
```
