# 12. Logging & Analysis

## 1. Crash Reporting
- **Tool:** Firebase Crashlytics.
- **Instruction to Dev:** Integrate Firebase BoM and Crashlytics in `build.gradle`. Ensure that any exceptions in database read/write are logged.

## 2. Analytics (Events to Track)
To understand user behavior, the developer should log the following custom events to Firebase Analytics:

1. `category_clicked`:
   - Parameter: `category_name` (e.g., "Love", "Attitude")
   - Purpose: To see which category is most popular.
2. `caption_copied`:
   - Purpose: To track how often the core feature is used.
3. `caption_shared`:
   - Purpose: To track virality actions.
4. `caption_favorited`:
   - Purpose: To track engagement.

### Dev Example:
```kotlin
val bundle = Bundle().apply {
    putString("category_name", category.name)
}
firebaseAnalytics.logEvent("category_clicked", bundle)
```
