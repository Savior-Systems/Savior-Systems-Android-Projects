# 09. Testing Assurance Plan

## 1. Developer Checklist Before Release
Rookie Developer MUST manually test these flows before making an APK.

### Flow 1: Database Initialization
- [ ] Install the app on a fresh emulator.
- [ ] Verify that categories load immediately on the Home screen.
- [ ] Verify that tapping "Attitude" shows a populated list of captions, not an empty screen.

### Flow 2: Core Actions
- [ ] Tap the "Copy" icon. Paste it in a notepad. Verify the exact text was copied.
- [ ] Tap the "Share" icon. Verify the Android Share Sheet opens with the text populated.
- [ ] Tap the "Heart" icon. Verify it changes color.

### Flow 3: Persistence
- [ ] Go to Favorites screen. Verify the liked caption is there.
- [ ] Kill the app (swipe away from recent apps).
- [ ] Reopen the app. Go to Favorites. Verify the caption is STILL there (tests Room DB persistence).

### Flow 4: Ad Functionality
- [ ] Ensure AdMob test IDs are used during development (DO NOT click your own real ads).
- [ ] Verify banner ad loads at the bottom of the home screen.
