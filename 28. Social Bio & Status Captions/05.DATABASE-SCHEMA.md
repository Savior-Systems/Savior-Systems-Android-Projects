# 05. Database Schema & Storage

## 1. Room Database Overview
We will have two tables: `categories` and `captions`. They have a one-to-many relationship (One category has many captions).

## 2. Tables & Columns

### Table 1: `categories`
| Column Name | Type    | Constraints                  | Description |
|-------------|---------|------------------------------|-------------|
| `id`        | Int     | Primary Key, Auto-Increment  | Unique ID   |
| `name`      | String  | Not Null                     | e.g. "Attitude", "Love" |
| `colorHex`  | String  | Not Null                     | e.g. "#FF0000" |
| `iconName`  | String  | Not Null                     | e.g. "ic_fire" |

### Table 2: `captions`
| Column Name   | Type    | Constraints                              | Description |
|---------------|---------|------------------------------------------|-------------|
| `id`          | Int     | Primary Key, Auto-Increment              | Unique ID   |
| `categoryId`  | Int     | Foreign Key -> categories(id), Not Null  | Link to Category |
| `text`        | String  | Not Null                                 | The caption itself |
| `isFavorite`  | Boolean | Default: 0 (False)                       | Is it bookmarked? |

## 3. Database Pre-population Strategy (For Rookie Dev)
**How to fill the database initially:**
Do not hardcode massive lists in the UI. 
Instead, create an `insertInitialData()` function in your `AppDatabase` callback or your Repository that runs ONCE when the app is first installed. Check `if (dao.getCategoryCount() == 0)`, and if true, insert the 50 hardcoded captions.

## 4. Required DAO Queries
```sql
-- Get all categories
SELECT * FROM categories

-- Get captions for a specific category
SELECT * FROM captions WHERE categoryId = :categoryId

-- Get all favorite captions
SELECT * FROM captions WHERE isFavorite = 1

-- Toggle favorite status (Update query)
UPDATE captions SET isFavorite = :isFav WHERE id = :id
```
