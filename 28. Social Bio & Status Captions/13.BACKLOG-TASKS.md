# 13. Step-by-Step Implementation Guide (Rookie Developer Roadmap)

**Dear Rookie Developer,**
Follow these exact steps in order. Do not skip ahead.

### Phase 1: Project Setup (Day 1)
1. Open Android Studio -> Create New Project -> "Empty Activity" (Jetpack Compose).
2. Open `build.gradle.kts` (Module: app). Add dependencies for Room, Hilt, Navigation Compose, and ViewModel Compose. Click "Sync Now".
3. Setup `strings.xml` and `colors.xml` (or `Color.kt` for Compose) based on the UI/UX Document (02).

### Phase 2: Database & Architecture (Day 1-2)
1. Create `CategoryEntity` and `CaptionEntity` data classes. Annotate them with `@Entity`.
2. Create `CaptionDao` interface with `@Query` functions for fetching and updating.
3. Create `AppDatabase` extending `RoomDatabase`.
4. Create a `Prepopulation.kt` file containing 50 quotes. Write logic in your `Repository` to insert them if `dao.getCategoryCount() == 0`.
5. Setup Hilt Modules to provide the Database and DAO.

### Phase 3: ViewModels & Logic (Day 2)
1. Create `MainViewModel` extending `ViewModel`. Annotate with `@HiltViewModel`.
2. Create `StateFlow` variables to hold the categories list, the active captions list, and the favorites list.
3. Write functions: `loadCaptions(categoryId)`, `toggleFavorite(caption)`, etc.

### Phase 4: UI Implementation (Day 3)
1. Create `HomeScreen.kt`. Use a `LazyVerticalGrid` to display the categories.
2. Create `CaptionListScreen.kt`. Use a `LazyColumn` for captions.
3. Add the Copy icon. Use Android's `ClipboardManager`. Add a `Toast` saying "Copied!".
4. Add the Share icon. Use `Intent.ACTION_SEND`.

### Phase 5: Polish & Monetization (Day 4)
1. Add Navigation Host in `MainActivity` to link the screens together.
2. Test the app extensively on an emulator (Follow 09. Testing Assurance Plan).
3. Integrate Google AdMob SDK. Add the Banner ID to the bottom of the screens.
4. Build Signed APK / AAB.
