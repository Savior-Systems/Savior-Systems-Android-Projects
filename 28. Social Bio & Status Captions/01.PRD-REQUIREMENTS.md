# 01. Product Requirements Document (PRD)

## 1. Project Context & Vision
**App Name:** Bio Status Captions Creator  
**Category:** Entertainment / Personalization  
**Core Vision:** To provide an offline-first, extremely lightweight, and fast application where users can discover, copy, and share high-quality bio templates and status captions for social media platforms (Instagram, WhatsApp, Facebook, TikTok). 

## 2. Target Audience
- Gen-Z and Millennials looking for aesthetic, attitude, or emotional captions for their daily posts.
- Users who want an offline repository without needing an active internet connection.

## 3. Scope of MVP (Minimum Viable Product)
For the MVP, we will keep the feature set small but perfectly executed. 
### In-Scope Features:
1. **Offline Pre-populated Database:** The app must ship with at least 50+ high-quality captions divided into 5 core categories (Attitude, Love, Motivation, Funny, Sad).
2. **Category Grid Navigation:** A visual home screen displaying categories.
3. **Caption Listing & Actions:** 
   - A list view of captions for the selected category.
   - **Copy to Clipboard:** 1-tap copy functionality with a Toast/Snackbar confirmation.
   - **Share:** 1-tap Android native share intent to send text directly to WhatsApp/Instagram.
4. **Favorites System:** Users can bookmark their favorite captions and view them in a dedicated "Favorites" screen.
5. **Dark Mode / Light Mode Support:** The UI must adapt to system theme preferences.

### Out-of-Scope (Future Updates):
- Cloud syncing of favorites.
- User accounts / Login.
- User submission of new captions.
- Image caption generation (Text-to-Image).

## 4. Rookie Developer Guidelines
- **Focus on Core Functionality First:** Do not worry about ads or complex animations until the core database and UI flows (Copy/Share/Favorite) are working flawlessly.
- **Strictly Follow Architecture:** Use MVVM. Do not put business logic or database queries inside the UI layers (Fragments/Activities/Compose).
