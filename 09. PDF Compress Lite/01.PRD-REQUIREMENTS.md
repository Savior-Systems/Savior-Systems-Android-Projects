# 01. Product Requirement Document (PRD) — PDF Compress Lite
`
## 1. Vision & Strategy
*   **App Name**: PDF Compress Lite
*   **Category**: Productivity / Utilities
*   **Core Objective**: Build a fast, lightweight, 100% offline-first application that allows users to shrink the file size of PDF documents (especially scanned documents) through rasterization and high-ratio image re-compression before uploading.
*   **Target Market**: Students and administrative applicants looking to meet file size limits on public services websites (often under 200KB or 1MB) securely on mobile.
`
---
`
## 2. Target Personas
`
### Persona A: Anisur Rahman (29, Barisal, Bangladesh)
*   **Role**: Government Job Applicant.
*   **Pain Point**: Local job portals require scanned certificates to be uploaded in PDF format with a strict limit of 300KB. His scanned PDF is currently 4.5MB.
*   **Usage**: Uses PDF Compress Lite to compress the document down to 250KB in seconds without internet exposure.
`
### Persona B: Jessica Miller (33, Chicago, USA)
*   **Role**: Self-Employed Accountant.
*   **Pain Point**: Wants to email client receipt bundles but the combined PDF file exceeds email attachment limits. Refuses to upload financial records to online compilers due to security.
*   **Usage**: Employs the app's local processing to shrink tax PDF file sizes quickly on her device.
`
---
`
## 3. Core Feature List & User Stories
`
### Feature 1: PDF Document Selector & Metadata Reader
*   **Requirement**: Import local PDF files via the Android Storage Access Framework. Read and display document metadata (page count, name, and current file size).
*   **User Story**: *As Anisur, I want to import my scanned certificate PDF and see its current file size clearly so that I know how much compression is required.*
`
### Feature 2: Adjustable Compression Quality Slider
*   **Requirement**: A slider offering three compression presets: Low (highest quality, ~80% size), Medium (balanced, ~50% size), and High (maximum compression, ~20% size).
*   **User Story**: *As Jessica, I want to adjust the compression slider so that I can balance legibility with file size optimization.*
`
### Feature 3: Local Bitmap Rasterization Engine
*   **Requirement**: Convert pages into bitmaps using native `PdfRenderer`, compress the bitmaps using JPEG compression at varying quality levels, and repackage them into a new output PDF completely offline.
*   **User Story**: *As Anisur, I want the compression to happen completely offline on my phone so that my private documents are never sent to third-party servers.*
`
---
`
## 4. Key Performance Indicators (KPIs)
*   **Cold Start Time**: < 400ms.
*   **Compression Velocity**: Compress a 10-page scanned document in < 5 seconds.
*   **Offline Sandboxing**: Zero network operations performed during document processing.
