# 02. UI/UX Design System — PDF Compress Lite
`
Clean, professional tool interface utilizing the Crimson Red color palette.
`
---
`
## 1. Color System (Crimson Red)
`
### Light Theme
*   **Primary (`#C62828`)**: Vivid Crimson Red for compression controls, sliders, and primary buttons.
*   **Primary Container (`#FFEBEE`)**: Soft pinkish-red container backing for status messages.
*   **Secondary (`#E53935`)**: Bright red accents.
*   **Tertiary (`#2E7D32`)**: Green highlighting compression savings.
*   **Background (`#FAFAFA`)**: Neutral light utility gray.
*   **Surface (`#FFFFFF`)**: Card panels for document statistics.
*   **On-Background (`#212121`)**: Clean dark gray text.
`
### Dark Theme
*   **Primary (`#FF8A80`)**: Pastel red accent to prevent blinding glow.
*   **Primary Container (`#C62828`)**: Deep crimson container backings.
*   **Secondary (`#FF5252`)**: Warning/Alert highlights.
*   **Tertiary (`#81C784`)**: Pastel green for savings data.
*   **Background (`#121212`)**: Dark slate background.
*   **Surface (`#1E1E1E`)**: Elevated cards.
*   **On-Background (`#E0E0E0`)**: Soft gray text.
`
---
`
## 2. Typography Standard
*   **Font Family**: **Outfit** (via Google Fonts Integration).
*   **Display Medium (Savings Percentage)**: 36sp, Bold, tracking -0.25px. Used to highlight the reduction size (e.g. "Saved 74%").
*   **Title Medium (File Details)**: 16sp, Semi-Bold.
`
---
`
## 3. Micro-Animations
`
### Animation A: Compression Progress Loader
*   **Trigger**: Compression logic runs.
*   **Behavior**: A circular progress indicator rotates with a spring spec (`tween` over 1500ms, infinite linear loop), while the file size numbers count down dynamically.
`
### Animation B: Savings Card Pop
*   **Trigger**: Compression completed.
*   **Behavior**: The "Compression Complete" card performs a tactile pop (`scaleY` & `scaleX` from 0.8 to 1.05 to 1.0) over 400ms using a standard spring bounce.
`
