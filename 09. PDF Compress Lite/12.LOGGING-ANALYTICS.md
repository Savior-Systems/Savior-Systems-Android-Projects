# 12. Logging & Analytics Schema — PDF Compress Lite

Tracking performance stability and optimization efficiency without violating user data boundaries.

---

## 1. Firebase Custom Events Schema

### Event A: `compress_task_complete`
*   **Trigger**: Compression resolves successfully.
*   **Parameters**:
    *   `original_size_kb` (integer)
    *   `compressed_size_kb` (integer)
    *   `compression_preset` (string: "low", "medium", "high")

### Event B: `compress_task_failed`
*   **Trigger**: Compression error occurs.
*   **Parameters**:
    *   `failure_reason` (string: "oom", "io_exception", "invalid_format")

### Event C: `history_cleared`
*   **Trigger**: User purges history list.
*   **Parameters**:
    *   `items_count` (integer)

### Event D: `consent_form_dismissed`
*   **Trigger**: GDPR popup handles input.
*   **Parameters**:
    *   `user_accepted` (integer: 1 or 0)

---

## 2. Strict Privacy Guardrails ("NEVER Log" Rules)
*   🚫 **NEVER log File Names**: File descriptors or user file names must never be recorded in parameters.
*   🚫 **NEVER log Document Streams**: PDF content bytes must never leave the local sandboxed memory.
*   🚫 **NEVER log File Contents**: Extracting or uploading text, images, or metadata summaries from the target document is strictly prohibited.
