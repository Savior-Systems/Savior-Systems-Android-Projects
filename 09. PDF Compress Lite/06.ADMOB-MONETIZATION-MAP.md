# 06. AdMob Monetization Map — PDF Compress Lite

Monetization maps ensuring ads do not block processing screens or trigger accidental clicks.

---

## 1. Ad Format Placements

| Format | Screen Target | Placement Details |
| :--- | :--- | :--- |
| **Banner** | Dashboard Bottom / History Logs | Anchored below options sliders. |
| **Native Ad** | History List Layout | Integrated after every 5 history items. |
| **Interstitial** | Compression Success Screen | Loaded and triggered upon completion of the compression task. |

---

## 2. Zero-Ad Zones
*   **Compression Progress Screen**: No banner or interstitial ads are loaded while the progress indicator is spinning.
*   **GDPR Consent Screen**: Complete block of ad SDK initialization before consent parameters resolve.

---

## 3. Strict Interstitial Cooldown

To avoid annoying users during multiple document compressions:
```kotlin
object CompressAdManager {
    private const val AD_COOLDOWN_MS = 180_000L // 3 minutes
    private var lastShowTimestamp = 0L

    fun shouldShowInterstitial(): Boolean {
        return (System.currentTimeMillis() - lastShowTimestamp) >= AD_COOLDOWN_MS
    }

    fun trackAdShown() {
        lastShowTimestamp = System.currentTimeMillis()
    }
}
```
*   **Usage**: On compression task complete, if `shouldShowInterstitial()` is true, trigger the interstitial; otherwise, transition immediately to the savings details.
