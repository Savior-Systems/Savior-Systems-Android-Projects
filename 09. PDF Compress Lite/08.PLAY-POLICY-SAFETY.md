```
## 1. System Permissions Audit
```
### Required Permissions
*   No runtime permissions requested. The app accesses files strictly via the Android Storage Access Framework (SAF) system picker, which operates outside of dangerous runtime permission scopes.
```
### Stripped / Unused Permissions (🚫 Zero Tolerance)
*   🚫 `android.permission.READ_EXTERNAL_STORAGE` (Not required when using SAF URI pickers)
*   🚫 `android.permission.WRITE_EXTERNAL_STORAGE` (Not required)
*   🚫 `android.permission.ACCESS_FINE_LOCATION` (Zero tracking)
*   🚫 `android.permission.INTERNET` (Requested only for AdMob SDK operations)
```

```
## 2. Play Console Data Safety Questionnaire
```
All compression algorithms run in-memory inside the sandboxed JVM heap.
```
| Section Question | Answer Status | Rationale |
| :--- | :--- | :--- |
| **Does your app collect or share user data?** | **No** | Documents are read from and written directly to the device storage via SAF. No file data is captured. |
| **Is all user data encrypted in transit?** | **N/A** | No data is sent over the network. |
| **Can users delete their history?** | **Yes** | History logs can be wiped via the Settings menu or individual item swipes. |
```
