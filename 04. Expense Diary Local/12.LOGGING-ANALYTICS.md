# 12. Logging & Analytics — Expense Diary Local
`
This document defines the Firebase Analytics telemetry schema, custom event parameters, and Crashlytics configuration for app optimization without collecting PII.
`
---
`
## 1. Firebase Analytics Configuration
`
### 1.1 SDK Integration
*   **BOM Version**: Firebase BOM 33+ (latest stable).
*   **Dependencies**: `firebase-analytics-ktx`, `firebase-crashlytics-ktx`.
*   **Consent**: Analytics collection starts only after GDPR/consent check (EU users). For non-EU users, collection begins on first app launch.
`
### 1.2 Automatic Events (No Configuration Needed)
Firebase automatically logs these events:
*   `first_open` — First app launch after install.
*   `session_start` — Beginning of a new session (after 30min inactivity).
*   `app_update` — First launch after app version update.
*   `os_update` — First launch after OS version change.
`

## 2. Custom Firebase Events
`
### 2.1 `transaction_added`
Triggered whenever a user saves a new transaction (expense or income).
*   **Parameters:**
    *   `transaction_type`: String (`"expense"` or `"income"`)
    *   `category_name`: String (e.g., `"food_dining"`, `"transport"`)
    *   `amount_range`: String (bucketed: `"0-100"`, `"100-500"`, `"500-1000"`, `"1000-5000"`, `"5000+"`)
    *   `entry_time_seconds`: Double (time from sheet open to save tap)
*   **Purpose**: Measure entry speed, popular categories, and spending ranges.
`
### 2.2 `transaction_deleted`
Triggered when a user deletes a transaction.
*   **Parameters:**
    *   `was_undone`: Integer (1 = undo snackbar tapped, 0 = permanent delete)
*   **Purpose**: Track deletion patterns and undo usage.
`
### 2.3 `budget_set`
Triggered when a user creates or updates a budget limit.
*   **Parameters:**
    *   `budget_type`: String (`"overall"` or `"category"`)
    *   `budget_amount_range`: String (bucketed: `"0-5000"`, `"5000-15000"`, `"15000-50000"`, `"50000+"`)
    *   `rollover_enabled`: Integer (0 or 1)
*   **Purpose**: Understand budget adoption and typical limit ranges.
`
### 2.4 `budget_alert_triggered`
Triggered when a budget threshold notification fires.
*   **Parameters:**
    *   `alert_type`: String (`"warning_80"` or `"exceeded_100"`)
    *   `category_name`: String (or `"overall"`)
    *   `overspend_percentage`: Integer (e.g., 105 means 5% over budget)
*   **Purpose**: Measure budget discipline and alert effectiveness.
`
### 2.5 `analytics_viewed`
Triggered when the user navigates to the Analytics dashboard.
*   **Parameters:**
    *   `date_range_selected`: String (`"this_month"`, `"last_30_days"`, `"custom"`)
    *   `chart_type_interacted`: String (`"pie"`, `"bar"`, `"none"`)
*   **Purpose**: Track analytics engagement and preferred visualizations.
`
### 2.6 `data_exported`
Triggered when the user exports data.
*   **Parameters:**
    *   `export_format`: String (`"csv"`, `"pdf"`, `"json_backup"`)
    *   `transaction_count`: Integer (number of records exported)
*   **Purpose**: Measure export feature adoption and preferred formats.
`
### 2.7 `app_lock_action`
Triggered when security settings change.
*   **Parameters:**
    *   `lock_type`: String (`"pin"`, `"biometric"`, `"disabled"`)
    *   `action`: String (`"enabled"`, `"disabled"`, `"changed"`)
*   **Purpose**: Track security adoption rates.
`
### 2.8 `theme_changed`
Triggered when the user switches the app theme.
*   **Parameters:**
    *   `new_theme`: String (`"system"`, `"light"`, `"dark"`)
*   **Purpose**: Understand dark mode adoption.
`
### 2.9 `currency_changed`
Triggered when the user selects a different currency.
*   **Parameters:**
    *   `currency_code`: String (ISO 4217, e.g., `"INR"`, `"USD"`, `"BDT"`)
*   **Purpose**: Track geographic usage distribution.
`

## 5. Crashlytics Configuration
`
### 5.1 Crash Reporting
*   **Automatic**: All uncaught exceptions are captured and uploaded on next session.
*   **Non-Fatal Logging**: Use `Firebase.crashlytics.recordException(e)` for caught exceptions in critical paths (DB operations, export, encryption).
`
### 5.2 Custom Keys for Crash Context
```kotlin
Firebase.crashlytics.setCustomKey("db_encrypted", true)
Firebase.crashlytics.setCustomKey("transaction_count", transactionCount)
Firebase.crashlytics.setCustomKey("last_screen", currentRoute)
Firebase.crashlytics.setCustomKey("app_lock_type", lockType)
```
`
### 5.3 Breadcrumbs
Log non-fatal breadcrumbs for debugging crash context:
```kotlin
Firebase.crashlytics.log("User opened add transaction sheet")
Firebase.crashlytics.log("Budget alert fired: 80% threshold for category=food")
Firebase.crashlytics.log("CSV export initiated with 150 transactions")
```

## 6. Privacy Compliance
`
### 6.1 No PII Collection Rules
*   ❌ Never log actual transaction amounts (use bucketed ranges).
*   ❌ Never log transaction notes or descriptions.
*   ❌ Never log category names created by users (use `"custom"` placeholder).
*   ❌ Never log PIN values or biometric data.
*   ✅ All analytics data is anonymous and aggregated.
`
### 6.2 Data Retention
*   Firebase Analytics: 14-month rolling retention (Google default).
*   Crashlytics: 90-day crash log retention.
*   No server-side data storage beyond Firebase.
