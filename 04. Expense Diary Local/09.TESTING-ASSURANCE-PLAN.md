```
## 4. Pre-Submission Manual Checks
```
### 4.1 Core Functionality
- [ ] **Orientation Change**: Verify transaction entry values survive rotation (SavedStateHandle).
- [ ] **Process Death Recovery**: Kill app via ADB during transaction entry; reopen and verify draft is preserved.
- [ ] **Offline Functionality**: Toggle airplane mode → confirm all features operate without network.
- [ ] **Large Dataset**: Insert 1,000+ transactions → verify smooth scrolling (Paging 3) and fast chart rendering.
```
### 4.2 Ad System
- [ ] **Banner Load**: Confirm adaptive banner loads with test unit ID on Dashboard.
- [ ] **Interstitial Timing**: Add 5 transactions rapidly → verify interstitial appears after 5th save.
- [ ] **Cooldown Enforcement**: After interstitial, add 5 more within 3 minutes → verify no second interstitial.
- [ ] **Ad-Free Zones**: Navigate to Settings, Budget Config, Security → verify zero ad presence.
```
### 4.3 Security
- [ ] **PIN Lock**: Set 4-digit PIN → force-close → reopen → PIN gate appears.
- [ ] **Biometric Lock**: Enable fingerprint → verify prompt on app launch.
- [ ] **Privacy Screen**: Enable privacy blur → switch to another app → return → verify blur overlay shown before auth.
- [ ] **Encrypted DB**: Use DB Browser to attempt opening `.db` file → confirm it's unreadable without passphrase.
```
### 4.4 Data Export
- [ ] **CSV Export**: Export with 50+ transactions → open in Excel → verify all fields present and formatted.
- [ ] **PDF Report**: Generate monthly PDF → verify charts, totals, and category breakdown are rendered.
- [ ] **JSON Backup**: Create backup → Clear all data → Restore from backup → verify data integrity.
```
