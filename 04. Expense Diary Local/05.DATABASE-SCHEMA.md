# 05. Database Schema & Storage — Expense Diary Local
`
This document details the SQLite schemas used in the Room Database, including data indexes, relationships, and preferences mappings.
`
---
`
## 1. Room Database Tables
`
### 1.1 Transactions Table (`transactions_table`)
The core table holding all income and expense records.
`
| Field Name | Data Type | Key Type | Constraints | Notes |
| :--- | :--- | :--- | :--- | :--- |
| `id` | INTEGER | Primary Key | Auto-Generate | Unique transaction identity. |
| `amount` | REAL | Non-Null | > 0.0 | Transaction amount in user's base currency. |
| `type` | TEXT | Non-Null | `EXPENSE` or `INCOME` | Transaction direction enum. |
| `category_id` | INTEGER | Foreign Key | References `categories_table(id)` | Associated category. |
| `note` | TEXT | Nullable | Max 200 chars | Optional user description. |
| `date` | INTEGER | Non-Null | Epoch millisecond | Transaction date (stored as Long). |
| `created_at` | INTEGER | Non-Null | Epoch millisecond | Record creation timestamp. |
| `updated_at` | INTEGER | Non-Null | Epoch millisecond | Last modification timestamp. |
`
### 1.2 Categories Table (`categories_table`)
Stores both preset and user-defined spending categories.
`
| Field Name | Data Type | Key Type | Constraints | Notes |
| :--- | :--- | :--- | :--- | :--- |
| `id` | INTEGER | Primary Key | Auto-Generate | Unique category identity. |
| `name` | TEXT | Non-Null | UNIQUE, Max 30 chars | Category display name (e.g., "Food & Dining"). |
| `icon_name` | TEXT | Non-Null | — | Material Icon identifier (e.g., `restaurant`). |
| `color_hex` | TEXT | Non-Null | `#RRGGBB` format | Category accent color for charts. |
| `is_preset` | INTEGER | Non-Null | 0 or 1 | Boolean: 1 = system default, 0 = user-created. |
| `sort_order` | INTEGER | Non-Null | — | Display ordering index. |
`
### 1.3 Budgets Table (`budgets_table`)
Stores monthly and per-category budget limits.
`
| Field Name | Data Type | Key Type | Constraints | Notes |
| :--- | :--- | :--- | :--- | :--- |
| `id` | INTEGER | Primary Key | Auto-Generate | Unique budget identity. |
| `category_id` | INTEGER | Foreign Key | Nullable, References `categories_table(id)` | NULL = overall monthly budget. |
| `amount_limit` | REAL | Non-Null | > 0.0 | Maximum spending threshold. |
| `month` | TEXT | Non-Null | `YYYY-MM` format | Budget period identifier. |
| `rollover_enabled` | INTEGER | Non-Null | 0 or 1 | Carry unspent budget forward. |
`

## 3. Indices & Constraints
`
### 3.1 Foreign Key Constraints
```sql
FOREIGN KEY(category_id) REFERENCES categories_table(id) ON DELETE SET NULL
-- When a custom category is deleted, associated transactions retain their data
-- but category_id is set to NULL (displayed as "Uncategorized").
`
FOREIGN KEY(category_id) REFERENCES categories_table(id) ON DELETE CASCADE
-- Budget entries: when a category is deleted, its budget entry is also removed.
```
`
### 3.2 Database Indices
```sql
CREATE INDEX idx_transactions_date ON transactions_table(date DESC);
-- Optimizes chronological listing and date-range queries.
`
CREATE INDEX idx_transactions_category ON transactions_table(category_id);
-- Optimizes category-based aggregation for analytics.
`
CREATE INDEX idx_transactions_type ON transactions_table(type);
-- Optimizes income vs. expense filtering.
`
CREATE INDEX idx_budgets_month ON budgets_table(month);
-- Optimizes monthly budget lookups.
`
CREATE UNIQUE INDEX idx_budgets_unique ON budgets_table(category_id, month);
-- Prevents duplicate budget entries for the same category-month combination.
```

## 4. Key DAO Queries
`
### 4.1 TransactionDao
```kotlin
@Query("SELECT * FROM transactions_table ORDER BY date DESC")
fun getAllTransactions(): PagingSource<Int, TransactionEntity>
`
@Query("SELECT * FROM transactions_table WHERE date BETWEEN :startDate AND :endDate ORDER BY date DESC")
fun getTransactionsByDateRange(startDate: Long, endDate: Long): Flow<List<TransactionEntity>>
`
@Query("SELECT SUM(amount) FROM transactions_table WHERE type = 'EXPENSE' AND date BETWEEN :startDate AND :endDate")
fun getTotalExpenseForPeriod(startDate: Long, endDate: Long): Flow<Double?>
`
@Query("SELECT SUM(amount) FROM transactions_table WHERE type = 'INCOME' AND date BETWEEN :startDate AND :endDate")
fun getTotalIncomeForPeriod(startDate: Long, endDate: Long): Flow<Double?>
`
@Query("SELECT category_id, SUM(amount) as total FROM transactions_table WHERE type = 'EXPENSE' AND date BETWEEN :start AND :end GROUP BY category_id ORDER BY total DESC")
fun getSpendingByCategory(start: Long, end: Long): Flow<List<CategorySpending>>
`
@Query("SELECT * FROM transactions_table WHERE note LIKE '%' || :query || '%' ORDER BY date DESC")
fun searchTransactions(query: String): Flow<List<TransactionEntity>>
```
`
### 4.2 BudgetDao
```kotlin
@Query("SELECT * FROM budgets_table WHERE month = :month")
fun getBudgetsForMonth(month: String): Flow<List<BudgetEntity>>
`
@Query("SELECT * FROM budgets_table WHERE category_id IS NULL AND month = :month LIMIT 1")
fun getOverallBudget(month: String): Flow<BudgetEntity?>
`
@Insert(onConflict = OnConflictStrategy.REPLACE)
suspend fun upsertBudget(budget: BudgetEntity)
```

## 5. DataStore Preferences
`
Preferences are managed via Jetpack DataStore (Proto-less, key-value):
`
| Key | Data Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `theme_mode` | INT | 0 | 0 = System, 1 = Light, 2 = Dark |
| `currency_code` | STRING | `"USD"` | ISO 4217 currency code |
| `currency_symbol` | STRING | `"$"` | Display symbol |
| `first_day_of_week` | INT | 1 | 1 = Monday, 7 = Sunday |
| `is_app_lock_enabled` | BOOLEAN | false | PIN / Biometric gate toggle |
| `app_lock_pin_hash` | STRING | `""` | SHA-256 hash of the user's PIN |
| `has_completed_onboarding` | BOOLEAN | false | First-launch gate flag |
| `privacy_screen_enabled` | BOOLEAN | true | Auto-blur on app background |
| `last_backup_timestamp` | LONG | 0 | Epoch ms of last JSON backup |
| `notification_80_enabled` | BOOLEAN | true | Budget 80% alert toggle |
| `notification_100_enabled` | BOOLEAN | true | Budget 100% alert toggle |
`

## 6. Migration Strategy
`
*   **Version 1 → 2 (Planned)**: Add `recurring_transaction_table` for auto-logged recurring expenses.
*   **Destructive Fallback**: Disabled. All migrations must be additive with `Migration(oldVersion, newVersion)` objects.
*   **Schema Export**: `exportSchema = true` in `@Database` annotation; schemas stored in `schemas/` directory for CI validation.
