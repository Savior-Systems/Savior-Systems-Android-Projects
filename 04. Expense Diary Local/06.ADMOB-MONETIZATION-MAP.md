# 06. AdMob Monetization Map — Expense Diary Local

This document defines the ad strategy, placement rules, frequency caps, and UX-safe zones for the Expense Diary Local application.

---

## 1. Monetization Strategy

Expense Diary Local uses a **freemium-light** approach: the app is fully functional for free, with non-intrusive ads that respect the user's financial workflow. Ads must never interrupt active data entry or obscure financial information.

---

## 2. Active Ad Formats

### 2.1 Adaptive Banner Ad
*   **Placement**: Anchored to the bottom of the Dashboard (Home) screen, above the bottom navigation bar.
*   **Size**: `AdSize.getCurrentOrientationAnchoredAdaptiveBannerAdSize()` — dynamically adjusts to screen width.
*   **Lifecycle**: Loaded on Dashboard `onResume`, paused on `onPause`, destroyed on `onDestroy`.
*   **Visibility Rules**:
    *   Shown **only** on the Dashboard screen.
    *   Hidden during active transaction entry (bottom sheet open).
    *   Hidden when the Analytics screen is in full-screen chart mode.

### 2.2 Interstitial Ad
*   **Trigger**: After every **5th transaction save** event (not on every save — user-friendly cadence).
*   **Frequency Cap**: Maximum 1 interstitial per **180 seconds** (3-minute cooldown).
*   **Preload**: Interstitial is preloaded during Dashboard initialization; reloaded after each display.
*   **Skip on Error**: If the ad fails to load, the flow continues without showing any ad.

### 2.3 Native Ad (Future V2)
*   **Planned Placement**: Inline between transaction groups in the history list (every 15th item).
*   **Status**: Not implemented in V1. Placeholder architecture reserved in `AdManager.kt`.

| Ad Format | Unit ID Source |
| :--- | :--- |
| Adaptive Banner | Stored in `BuildConfig.ADMOB_BANNER_ID` (injected via `local.properties`) |

**Security**: Production Ad Unit IDs are **never** hardcoded in source files. They are read from `local.properties` and injected at build time via `buildConfigField`.

## 4. UX-Safe Zones (No Ads Allowed)

The following screens and states are designated **ad-free zones** to ensure trust and policy compliance:

| Screen / State | Reason |
| :--- | :--- |
| **Settings Screen** | User is configuring preferences — no interruptions. |
| **Budget Configuration** | User is setting financial limits — sensitive workflow. |
| **Security Setup (PIN/Biometric)** | Authentication setup — must be distraction-free. |
| **Backup & Restore** | Data migration in progress — no blocking overlays. |
| **First Launch / Onboarding** | New user experience must be seamless. |
| **Transaction Entry Sheet** | Active data entry — never interrupt with ads. |

### 5.2 AdManager Singleton
```kotlin
@Singleton
class AdManager @Inject constructor(
    @ApplicationContext private val context: Context
) {
    private var interstitialAd: InterstitialAd? = null
    private var lastInterstitialShown: Long = 0
    private var transactionSaveCount: Int = 0
    
    private val INTERSTITIAL_COOLDOWN_MS = 180_000L  // 3 minutes
    private val TRANSACTION_TRIGGER_COUNT = 5
    
    fun onTransactionSaved(): Boolean {
        transactionSaveCount++
        if (transactionSaveCount % TRANSACTION_TRIGGER_COUNT == 0) {
            return showInterstitialIfReady()
        }
        return false
    }
    
    private fun showInterstitialIfReady(): Boolean {
        val now = System.currentTimeMillis()
        if (now - lastInterstitialShown < INTERSTITIAL_COOLDOWN_MS) return false
        // ... show interstitial logic
        return true
    }
}
```
