## 3. Localization Best Practices

### 3.1 String Formatting
*   Use `%1$s` for string parameters and `%1$d` for integers (positional parameters for translator reordering).
*   Never concatenate strings in code — use parameterized `getString()` calls.
*   All currency amounts must use `NumberFormat.getCurrencyInstance()` with the user's selected locale.

### 3.2 RTL Support
*   Use `android:supportsRtl="true"` in `AndroidManifest.xml`.
*   Replace all `marginLeft`/`paddingRight` with `marginStart`/`paddingEnd`.
*   Test the Arabic locale with "Force RTL Layout Direction" in Developer Options.

### 3.3 Date & Number Formatting
*   Dates: `DateTimeFormatter.ofLocalizedDate(FormatStyle.MEDIUM)` with user locale.
*   Numbers: `NumberFormat.getInstance(locale)` for grouping separators.
*   Currency: Respect `DecimalFormatSymbols` for locale-specific decimal points vs. commas.

### 3.4 Plurals Handling
```xml
<plurals name="transactions_count">
    <item quantity="one">%1$d transaction</item>
    <item quantity="other">%1$d transactions</item>
</plurals>
```
