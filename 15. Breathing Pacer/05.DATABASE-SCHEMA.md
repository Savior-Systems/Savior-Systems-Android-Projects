# 05. Database Schema
`
This document details storage properties for **Breathing Pacer**.
`
---
`
## 1. Storage Choice
Since Breathing Pacer is a zero-telemetry utility app, it does not require a local SQLite database (Room). Instead, it uses **Jetpack DataStore** to store preferences.
`
---
`
## 2. Preference Keys & Types
*   `pacer_mode`: `INTEGER` (0 = Box Breathing, 1 = Relax, 2 = Custom)
*   `inhale_duration`: `INTEGER` (Seconds, default `4`)
*   `hold_in_duration`: `INTEGER` (Seconds, default `4`)
*   `exhale_duration`: `INTEGER` (Seconds, default `4`)
*   `hold_out_duration`: `INTEGER` (Seconds, default `4`)
*   `haptic_feedback_enabled`: `BOOLEAN` (Vibrations at state boundaries, default `true`)
*   `keep_screen_on`: `BOOLEAN` (Default `true`)
`
