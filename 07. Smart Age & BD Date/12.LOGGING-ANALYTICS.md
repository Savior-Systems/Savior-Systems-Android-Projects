# 12. Logging & Analytics Schema — Smart Age & BD Date
`
Tracking app stability and layout performance without violating user privacy. No personally identifiable information (PII) is ever recorded.
`
---
`
## 1. Firebase Custom Events Schema
`
We track 4 custom analytics events to analyze user engagement:
`
### Event A: `calculate_age_run`
*   **Trigger**: User clicks "Calculate Age".
*   **Parameters**:
    *   `has_target_date` (integer: 1 or 0)
    *   `result_years` (integer: computed age years value)
`
### Event B: `date_diff_run`
*   **Trigger**: User performs date range calculations.
*   **Parameters**:
    *   `total_days_difference` (integer: computed days gap)
`
### Event C: `add_birthday_profile`
*   **Trigger**: User saves a contact's birthday.
*   **Parameters**:
    *   `has_notes` (integer: 1 or 0)
    *   `avatar_color_index` (integer)
`
### Event D: `ad_interaction`
*   **Trigger**: Ad layout clicked or dismissed.
*   **Parameters**:
    *   `ad_format` (string: "banner", "interstitial", "native")
`

`
## 2. Strict Privacy Guardrails ("NEVER Log" Rules)
`
To remain 100% compliant with privacy guidelines, the application enforces the following barriers:
*   🚫 **NEVER log Names**: Contact names saved in the registry must never be sent to logs or Firebase telemetry.
*   🚫 **NEVER log Exact DOB**: Do not log raw birthday dates (e.g. `1990-05-12`). Only aggregate integers (e.g., age years) are permitted parameters.
*   🚫 **NEVER log Location/IP**: Location telemetry must be fully stripped.
`
