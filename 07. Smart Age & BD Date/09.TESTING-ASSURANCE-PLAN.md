## 2. Manual QA Matrix (Edge Cases)
```
| QA Case | Action | Expected Behavior |
| :--- | :--- | :--- |
| **Leap Year DOB** | Set DOB as `2000-02-29`, target date as `2021-02-28`. | Age should output exactly 21 Years, 0 Months, 0 Days (not crash or offset). |
| **DST Shift** | Set calculation target across Autumn/Spring clock adjustments. | Age in hours/seconds must resolve correctly without timezone skew. |
| **Date Inversion** | Input DOB `2020-01-01` and Target Date `2010-01-01`. | Button must disable; error message: *"Target Date must be after Date of Birth"* must display. |
| **Notification Toggle** | Disable notifications in system settings, then attempt to save a profile. | Show snackbar explaining that birthday reminders require notifications permission. |
```

## 3. Performance & Cold Start Benchmarks
```
*   **Boot Speed**: The app must load its primary activity screen and display the calculator form in `< 400ms` from cold boot.
*   **Database Execution**: Profile queries must compile and return via Flow in under `15ms`.
*   **Memory Footprint**: App active heap memory usage must remain under `64MB` during standard workflows.
