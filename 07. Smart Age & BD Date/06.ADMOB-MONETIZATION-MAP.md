# 06. AdMob Monetization Map — Smart Age & BD Date
```
Monetization is designed to balance user experience and revenue. Ads are never displayed in ways that block primary utilities.

```
## 1. Ad Format Placements
```
| Format | Screen Target | Placement Details |
| :--- | :--- | :--- |
| **Banner** | Main Calculator / Date Diff | Anchored to the bottom, below calculation buttons. |
| **Native Ad** | Birthday Registry Screen | Integrated as a list item in the profiles list after every 4 profiles. |
| **Interstitial** | Calculator Action Result | Triggered when clicking "Calculate Age" *only* if calculations run. |
```
---
```
## 2. Zero-Ad Zones (Crucial for Policy Compliance)
*   **GDPR Consent Screen**: Zero ads or monetization hooks loaded until the user completes the UMP form.
*   **Add Profile Dialog**: No ads display on overlays where inputs are entered to prevent accidental clicks.
*   **App Launch & Boot**: No app-open ads or interstitials on boot to comply with the Google Play Store's disruptive ads policy.
```

## 3. Strict Interstitial Cooldown & Rate Limiting
```
To avoid user frustration and AdMob penalty:
```kotlin
object AdMonetizationManager {
    private const val INTERSTITIAL_COOLDOWN_MS = 180_000L // 3 minutes
    private var lastAdShownTimestamp = 0L
```
    fun shouldShowInterstitial(): Boolean {
        val currentTime = System.currentTimeMillis()
        val elapsed = currentTime - lastAdShownTimestamp
        return elapsed >= INTERSTITIAL_COOLDOWN_MS
    }
```
    fun recordAdShown() {
        lastAdShownTimestamp = System.currentTimeMillis()
    }
}
```
```
*   **Implementation**: Before showing the result sheet interstitial, check `shouldShowInterstitial()`. If false, skip the ad and render results immediately.
